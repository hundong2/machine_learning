# Beginner Review Log

요청에 맞춰 초보자 관점에서 전체 자료를 세 번 점검하고 보완한 기록입니다.

## 1회차: 용어 장벽 점검

확인한 문제:

- 논문 용어와 코드 용어가 섞여 있어, `patch`, `token`, `embedding`은 이해해도 `shape`, `axis`, `temperature`에서 막힐 수 있습니다.
- diffusion의 `alpha_bar`, MAE의 `mask_ratio`, CLIP의 `cosine similarity`처럼 논문마다 갑자기 등장하는 변수명이 초보자에게 부담이 됩니다.

보완한 내용:

- [glossary.md](./glossary.md)에 코드에서 자주 만나는 용어 섹션을 추가했습니다.
- 논문 개념과 노트북 변수명이 어떻게 연결되는지 표로 설명했습니다.

## 2회차: 코드 함수 장벽 점검

확인한 문제:

- 노트북에는 주석이 있지만, `np.array`, `reshape`, `@`, `np.linalg.norm`, `plt.imshow` 같은 기본 함수의 의미를 모르면 실행은 해도 이해가 얕아질 수 있습니다.
- 시각화 코드는 결과를 보여주지만, 어떤 그림을 어떻게 읽어야 하는지 별도 안내가 필요합니다.

보완한 내용:

- [code_function_guide.md](./code_function_guide.md)를 추가했습니다.
- 배열 만들기, 난수, 신경망 계산, 그림 그리기 함수들을 논문별 실습과 연결했습니다.

## 3회차: 학습 흐름 장벽 점검

확인한 문제:

- 자료를 다 읽어도 "내가 이해했는지" 판단할 질문이 없으면 수동적인 읽기에서 끝날 수 있습니다.
- 연구자 관점으로 넘어가려면 각 논문마다 스스로 반박하거나 재현할 질문이 필요합니다.

보완한 내용:
