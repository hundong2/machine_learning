# LeNet-5 논문 한국어 의역 자료

> 이 문서는 논문 원문 전체 번역이 아니라, 초보자가 논문의 흐름을 잡기 위한 한국어 의역과 해설입니다. 정확한 실험 조건, 표, 수식은 [원문](https://yann.lecun.com/exdb/publis/pdf/lecun-01a.pdf)을 기준으로 확인하세요.

## 논문 제목 의역

**문서 인식에 적용한 기울기 기반 학습**

## 한 문장 요약

이미지를 1차원 표로 펴지 말고, 작은 필터가 훑으며 2차원 모양 정보를 보존하자는 CNN의 출발점입니다.

## 초록을 초보자 말로 풀어쓰기

손글씨 우편번호나 수표 숫자는 같은 숫자라도 위치, 굵기, 기울기가 조금씩 다릅니다. 픽셀을 그대로 펼친 완전연결망은 이 공간적 변화를 다루기 어렵습니다.

작은 필터를 이미지 전체에 공유해 적용하는 합성곱과, 위치 변화에 둔감하게 만드는 풀링을 번갈아 쌓습니다. 마지막에는 분류기로 연결해 숫자를 예측합니다.

LeNet-5는 실제 문서 인식 시스템에 쓰일 만큼 안정적이었고, 오늘날 CNN의 기본 골격인 conv -> pooling -> classifier 흐름을 보여주었습니다.

## 핵심 주장 3개 의역

1. 이 논문은 기존 방법이 놓치던 병목을 문제로 정의합니다. 먼저 "왜 이 문제가 그 시점에 중요했는가"를 잡으면 좋습니다.
2. 제안 방법의 힘은 복잡한 장식보다 구조적 관점의 변화에서 나옵니다. 즉, 모델이 어떤 정보를 더 쉽게 보거나 전달하도록 길을 바꾼 것입니다.
3. 실험 결과는 단순히 점수가 좋아졌다는 뜻만이 아니라, 이후 연구자들이 어떤 방향으로 모델을 키우고 조합할지에 대한 기준점을 만들었습니다.

## 처음 읽을 때 표시할 부분

- 문제 정의: 논문이 무엇을 실패 사례로 보고 있는지 찾습니다.
- 모델 그림: 데이터가 어떤 모양으로 들어가 어떤 모양으로 나오는지 따라갑니다.
- 손실 함수 또는 학습 목표: 모델이 무엇을 맞히도록 훈련되는지 확인합니다.
- 실험 표: 단순 최고 점수보다, 어떤 조건을 바꿨을 때 성능이 바뀌는지 봅니다.

## 연구자처럼 던질 질문

- 이 논문의 가정은 지금도 유효한가?
- 모델이 잘되는 이유를 저자들은 실험으로 어떻게 분리했는가?
- 내가 더 작은 장난감 데이터로 같은 현상을 재현한다면 무엇을 측정해야 하는가?
