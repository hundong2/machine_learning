"왜 이미지를 볼 때 반드시 합성곱(CNN) 돋보기로 훑어야만 하는가? 이미지를 '글자(Text)'처럼 취급해서, 자연어를 정복한 트랜스포머(Transformer)에 그대로 밀어 넣으면 안 될까?"

이 대담하고도 불손한 질문이 어떻게 CNN의 10년 독재를 끝내고, 오늘날 생성형 AI와 거대 파운데이션 모델(Foundation Model)의 뼈대가 되었는지 그 우아한 수학적 철학을 내가 직접 해부해 주겠네.

---

## 1. 시대적 배경: 왜 CNN을 버려야 했는가? (The "Why")

CNN은 지난 10년간 비전 AI의 절대 군주였네. 그 강력함의 비결은 수학적으로 귀납적 편향(Inductive Bias)이 매우 강하게 설계되었다는 점이지.

* **지역성 (Locality):** 픽셀은 주변 픽셀과 가장 연관이 깊을 것이라는 가정 (3x3 돋보기 사용).
* **이동 불변성 (Translation Invariance):** 고양이가 사진의 왼쪽에 있든 오른쪽에 있든 똑같이 고양이라는 가정 (가중치 공유).

**[문제점의 발생]**
이러한 강한 가정(편향)은 데이터가 적을 때는 학습을 아주 빠르고 안정적으로 만들어 주었네. 하지만 구글, 메타 같은 기업들이 수십억 장의 이미지(JFT-300M 등)를 확보하게 된 시대가 오자, 오히려 이 '미리 정해둔 규칙'이 모델의 지능이 폭발적으로 확장(Scaling)되는 것을 가로막는 **유리천장**이 되어버렸네.

반면, 자연어 처리(NLP)를 정복한 트랜스포머는 수학적 가정이 거의 없는 순수한 구조였네. 데이터만 무한대로 쏟아부으면 스스로 세상의 규칙을 깨우쳤지. 그래서 학자들은 결심했네. **"가정을 버리자. 이미지를 문장으로 만들어서 트랜스포머에 던져주자!"**

## 2. ViT의 3단계 수학적 마법: 이미지를 언어로 바꾸다

---

ViT의 핵심은 "어떻게 2차원 이미지를 트랜스포머가 이해할 수 있는 1차원 시퀀스(Sequence, 문장)로 변환하는가?"에 있네.

### A. 패치 분할 (Patch Extraction & Flattening)

해상도가 $H \times W$ 이고 채널이 $C$ 인 이미지 $x \in \mathbb{R}^{H \times W \times C}$ 가 있네.
이 이미지를 $P \times P$ 크기의 작은 정사각형 조각(Patch)들로 자르네. (보통 $16 \times 16$ 픽셀)
그러면 잘린 패치의 총 개수 $N$ 은 다음과 같네.

$$N = \frac{H \times W}{P^2}$$

예를 들어 $224 \times 224$ 이미지를 $16 \times 16$ 으로 자르면, 총 $N = 196$ 개의 패치가 나오네. 즉, 사진 한 장이 **196개의 단어(Token)로 이루어진 하나의 문장**으로 변환된 것이지!

각 패치는 이제 가로 $P$, 세로 $P$, 채널 $C$ 를 곱한 길이를 가진 1차원 벡터로 평탄화(Flatten) 되네.


$$x_p \in \mathbb{R}^{N \times (P^2 \cdot C)}$$

### B. 선형 투영 (Linear Projection / Patch Embedding)

이 평탄화된 패치 벡터들은 아직 차원이 너무 커서 다루기 불편하네. 그래서 학습 가능한 가중치 행렬 $E \in \mathbb{R}^{(P^2 \cdot C) \times D}$ 를 곱하여, 트랜스포머가 좋아하는 일정한 차원 $D$ 로 압축하네.

이것이 언어 모델에서 단어를 임베딩(Word Embedding)하는 것과 완벽하게 동일한 수학적 역할을 하므로 **Patch Embedding**이라고 부르지.

### C. 위치 인코딩 (Positional Encoding)

트랜스포머는 수학적으로 순열 불변성(Permutation Invariant)을 가지네. 196개의 패치를 순서대로 주든, 섞어서 주든 내부 연산 결과가 똑같다는 뜻이지.
"이 패치는 사진의 왼쪽 위에 있던 조각이야!"라는 2차원적 공간 정보를 모델에게 알려주기 위해, 각 패치 임베딩 벡터에 학습 가능한 위치 정보 벡터 $E_{pos}$ 를 더해주네. (더하기 연산임에 주목하게!)

$$z_0 = [x_{class}; \; x_{p}^1 E; \; x_{p}^2 E; \; \cdots \; ; x_{p}^N E] + E_{pos}$$

* $x_{class}$: 오직 '이 사진이 무엇인지'를 분류하기 위해 맨 앞에 추가된 특수한 빈 상자(Token)라네. (BERT에서 빌려온 아이디어지).
* $z_0$: 트랜스포머의 첫 번째 층으로 들어가는 최종 입력 행렬.

## 3. 핵심 엔진: 자아-주의 메커니즘 (Self-Attention)

CNN이 $3 \times 3$ 돋보기로 바로 옆 픽셀만 쳐다볼 때(Local), ViT의 트랜스포머는 첫 번째 층부터 **196개의 패치 전체가 서로를 동시에 쳐다보며 연관성을 계산**하네(Global).

이것을 가능하게 하는 수학적 심장이 바로 **Multi-Head Self-Attention (MSA)** 이네.
입력 벡터 $X$에 세 개의 서로 다른 가중치 행렬을 곱해 쿼리($Q$, 질문), 키($K$, 답표), 밸류($V$, 정보)를 생성하네.

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q K^T}{\sqrt{d_k}}\right) V$$

* $Q K^T$: 1번 패치가 196개의 모든 패치와 얼마나 수학적으로 닮아있는지(내적, Dot Product)를 계산한 호감도 행렬이네. (예: '강아지 귀' 패치는 '강아지 코' 패치와 높은 값을 가짐).
* 이 호감도를 확률값(Softmax)으로 바꾸어 다른 패치들의 정보($V$)를 끌어모으네.
* 즉, 아무리 멀리 떨어져 있는 픽셀이라도 단 한 번의 행렬 곱셈으로 정보를 교환하는, **거리의 제약이 완전히 파괴된(Long-range Dependency) 아키텍처**인 것이지!

---

## 4. 핵심 용어 정리

* **ViT (Vision Transformer):** 이미지를 패치(조각) 단위로 잘라 시퀀스로 만든 뒤 트랜스포머 인코더에 통과시켜 분류하는 모델.
* **Patch (패치):** 이미지를 텍스트의 '단어(Token)'처럼 취급하기 위해 잘라낸 작은 정사각형 영역.
* **Inductive Bias (귀납적 편향):** 모델이 학습하지 않은 데이터에 대해 예측할 때 사용하는, 설계자가 미리 부여한 가정. (CNN은 높고, ViT는 낮음).
* **[CLS] Token (Class Token):** 모델이 196개의 패치를 다 살펴본 후, "결과적으로 이 사진 전체의 의미가 뭐야?"라는 최종 결론을 담아내는 특수 목적의 벡터.

## 5. 응용 분야: 비전 파운데이션 모델의 시대

ViT의 파급력은 그야말로 파괴적이었네. 데이터가 무한정 주어지면 성능이 끝없이 올라간다는 트랜스포머의 법칙(Scaling Law)이 비전 분야에서도 증명되었으니까!

1. **의료 및 위성 영상:** 수천 픽셀이 넘는 거대한 이미지에서, 멀리 떨어진 두 병변(암세포) 간의 상관관계를 파악할 때 Self-Attention의 강력함이 발휘되네.
2. **생성형 AI (Generative AI):** 최근 세상을 놀라게 한 OpenAI의 동영상 생성 AI인 Sora(소라)의 핵심 엔진이 바로 이 ViT 구조를 차용한 DiT (Diffusion Transformer)라네! (U-Net이라는 CNN 구조를 버려버렸지).
3. **자율주행 환경의 3D 공간 인식:** 카메라, 라이다(LiDAR) 등 여러 센서의 정보를 '토큰'으로 통일하여 하나의 트랜스포머에서 융합(Sensor Fusion)하는 구조가 표준이 되고 있네.

## 6. PyTorch 코드 예시: पै치 임베딩의 우아함

CNN을 버리고 이미지를 문장으로 바꾸는 가장 중요한 첫 단계인 **Patch Embedding**이 코드로 얼마나 아름답게 구현되는지 보게. 놀랍게도 PyTorch에서는 커널 크기와 스트라이드가 똑같은 `Conv2d`를 사용하여 이 패치 분할을 단 한 줄로 깔끔하게 처리하네.

```python
import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    def __init__(self, in_channels=3, patch_size=16, embed_dim=768, img_size=224):
        super(PatchEmbedding, self).__init__()
        self.patch_size = patch_size
        
        # 놀라운 트릭: 커널 크기와 스트라이드를 모두 패치 크기(16)로 동일하게 주면,
        # 이미지가 서로 겹치지 않는 16x16 패치 단위로 정확히 썰리게 됩니다!
        self.proj = nn.Conv2d(
            in_channels, 
            embed_dim, 
            kernel_size=patch_size, 
            stride=patch_size
        )
        
        # 총 패치 개수 N = (224/16) * (224/16) = 14 * 14 = 196
        self.num_patches = (img_size // patch_size) ** 2
        
        # 학습 가능한 특수 토큰 [CLS] 추가 (1, 1, 768)
        self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))
        
        # 위치 임베딩 정보 추가 (1, 196 + 1, 768) -> +1은 CLS 토큰의 자리
        self.pos_embed = nn.Parameter(torch.zeros(1, self.num_patches + 1, embed_dim))

    def forward(self, x):
        B = x.shape[0] # Batch Size
        
        # 1. 패치 자르기 및 투영: (B, 3, 224, 224) -> (B, 768, 14, 14)
        x = self.proj(x)
        
        # 2. 1차원 시퀀스로 평탄화: (B, 768, 196) -> (B, 196, 768)
        x = x.flatten(2).transpose(1, 2)
        
        # 3. 모든 배치의 맨 앞에 [CLS] 토큰 붙이기 -> (B, 197, 768)
        cls_tokens = self.cls_token.expand(B, -1, -1)
        x = torch.cat((cls_tokens, x), dim=1)
        
        # 4. 위치 정보 더해주기 (순서 부여)
        x = x + self.pos_embed
        
        # 이제 이 텐서는 순수한 언어(문장) 모델의 입력 형태와 완벽히 똑같습니다!
        # TransformerEncoder(x) 로 넘기면 끝납니다.
        return x

# --- 실험 ---
# 224x224 컬러 이미지 한 장 (Batch=1, C=3, H=224, W=224)
dummy_image = torch.randn(1, 3, 224, 224)
patch_embed_layer = PatchEmbedding()

output_tokens = patch_embed_layer(dummy_image)
print("트랜스포머에 들어가는 최종 텐서 크기:", output_tokens.shape)
# 출력: torch.Size([1, 197, 768]) (1개의 이미지, 197개의 토큰, 각 토큰은 768차원)

```
