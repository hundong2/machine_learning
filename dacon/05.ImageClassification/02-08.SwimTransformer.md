## Swin Transformer (Shifted Window Transformer)

2021년 마이크로소프트 아시아 연구소(MSRA)의 제 리우(Ze Liu) 연구팀이 발표한 이 알고리즘은, "ViT가 CNN을 파괴했다면, 왜 객체 탐지나 분할(Segmentation) 같은 밀집 예측(Dense Prediction)에서는 여전히 ViT를 쓰기 힘든가?"라는 본질적인 수학적 의문에서 출발했네.

그들이 어떻게 ViT의 수학적 모순을 해결하고 CNN의 우아한 계층 구조를 부활시켰는지, 그 경이로운 설계도를 함께 파헤쳐 보세.

---

## 1. ViT의 치명적 결함: 제곱 비례 연산량의 저주 (The "Why")

자네도 알다시피 ViT는 이미지 전체의 패치들이 서로를 쳐다보는 전역 자아-주의(Global Self-Attention)를 수행하네.
이미지의 가로 패치 수를 $w$, 세로 패치 수를 $h$라고 하면 총 패치의 개수 $N = hw$ 가 되지. 트랜스포머의 연산 복잡도(Computational Complexity) $\Omega$ 를 증명하는 수식은 다음과 같네.

$$\Omega(\text{Global MSA}) = 4hwC^2 + 2(hw)^2C$$

* $4hwC^2$: $Q, K, V$ 행렬을 만들고 마지막에 선형 변환하는 비용 (이것은 선형적이라 문제없네).
* **$2(hw)^2C$:** 이것이 문제의 핵심이네! $Q$와 $K^T$를 곱할 때 $(hw \times hw)$ 크기의 거대한 어텐션 행렬이 만들어지는데, 여기서 연산량이 이미지 크기의 제곱($(hw)^2$)에 비례하여 폭발하네.

**[현실적 재앙]**
고양이인지 개인지 맞추는 단순 분류(Classification)는 해상도가 $224 \times 224$면 충분하니 ViT도 버틸 수 있네. 하지만 픽셀 단위로 객체를 쪼개는 분할(Segmentation)이나 객체 탐지를 하려면 $1024 \times 1024$ 이상의 고해상도가 필요한데, 해상도가 4배 커지면 연산량은 **16배**가 아닌 **256배**로 폭발해버리지. GPU가 버틸 수 없는 구조였네.

---

## 2. 첫 번째 혁명: 윈도우 기반 어텐션 (W-MSA)

이 저주를 풀기 위해 Swin Transformer는 W-MSA (Window Multi-head Self-Attention)라는 개념을 도입하네.
"사진 전체를 다 보지 마라. 이미지를 $M \times M$ (보통 $M=7$) 크기의 고정된 격자 창문(Window)으로 나누고, 어텐션은 오직 그 창문 안에서만 수행하라!"

이 아주 간단한 제약이 수학적으로 얼마나 거대한 기적을 일으켰는지 보게.
하나의 윈도우 안에는 $M^2$ 개의 패치가 있네. 이 윈도우 안에서의 어텐션 연산량은 $2(M^2)^2 C$ 이지. 그런데 이런 윈도우가 사진 전체에 총 $\frac{hw}{M^2}$ 개가 존재하네. 두 개를 곱하면?

$$\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2hwC$$

자, 수식을 자세히 보게나. $(hw)^2$ 이라는 괴물이 사라지고, 상수 $M^2$ 이 붙은 **$hw$ 에 대한 선형(Linear) 방정식**으로 탈바꿈했네! 이제 이미지가 아무리 커져도 연산량은 정비례로만 증가하게 되었지.

---

## 3. 두 번째 혁명: 이동된 윈도우 (Shifted Window, SW-MSA)

W-MSA로 속도는 엄청나게 빨라졌지만, 치명적인 부작용이 생겼네. 창문(Window)을 고정해 버렸더니, 창문 밖의 이웃 픽셀들과는 정보 교환이 완전히 단절된 것이네. (마치 벽으로 막힌 방에 사람들을 가둔 것과 같지).

이를 해결하기 위해 고안된 천재적인 트릭이 바로 Shifted Window (이동된 창문)이네.
연속된 두 개의 트랜스포머 블록이 있을 때:

1. **층 $l$ (W-MSA):** 일반적인 $M \times M$ 격자로 나눈다.
2. **층 $l+1$ (SW-MSA):** 이 격자를 우측 하단으로 $\lfloor M/2 \rfloor$ 픽셀만큼 **이동(Shift)** 시킨다!

이렇게 창문을 반 칸씩 어긋나게 겹쳐서 다음 층을 설계하면, 이전 층에서는 서로 다른 방에 갇혀 있던 패치들이 새로운 방에서 만나 정보를 교환(Cross-window Connection)하게 되네. 전역 연산(Global Attention)을 하지 않고도 모델 전체에 정보가 흐르게 만든 것이지.

---

## 4. SW-MSA의 구현체: 순환 이동 (Cyclic Shift)과 마스킹 (Masking)

수학적 아이디어는 완벽했지만, 이것을 코드로 짜려니 심각한 문제가 발생했네.
창문을 우측 하단으로 이동시켰더니, 이미지의 위쪽과 왼쪽 가장자리에 크기가 $M \times M$ 보다 작은 자투리 창문들이 생겨버렸지. 윈도우의 개수도 늘어나버려 행렬 연산을 일괄적으로(Batch) 처리할 수 없게 되었네.

Swin 연구팀은 이를 해결하기 위해 순환 이동(Cyclic Shift)이라는 기가 막힌 엔지니어링 마법을 부렸네.

1. **잘라내기 (Slice):**
오른쪽과 아래쪽으로 밀려나서 이미지 밖으로 튀어나간 자투리 패치들(A, B, C 구역)을 잘라냅니다.


2. **순환 이동 (Cyclic Shift):**
잘라낸 이 조각들을 텅 비어있는 반대편 가장자리 (위쪽과 왼쪽)로 마치 팩맨(Pac-Man) 게임처럼 이동시켜 채워 넣습니다. 이렇게 하면 창문의 개수와 크기가 처음과 완벽하게 똑같아집니다.


3. **어텐션 마스킹 (Attention Masking):**
이 단계가 가장 중요하네! 순환 이동으로 억지로 붙여놓았지만, 사실 이 패치들은 원본 사진에서 서로 반대편 끝(예: 왼쪽 끝과 오른쪽 끝)에 있던 전혀 상관없는 픽셀들이네. 서로 어텐션 점수를 계산하지 못하도록, 이들이 만나는 경계선에 `-100이라는 음의 무한대 값`을 주어 소프트맥스(Softmax) 결과가 0이 되도록 마스킹(Masking)해 버리네!


4. **원위치 (Reverse Cyclic Shift):**
어텐션 연산이 끝난 후에는 다시 역방향으로 이동시켜 패치들을 원래 위치로 되돌려 놓습니다.

## 5. CNN의 계층적 구조 부활: 패치 병합 (Patch Merging)

ViT는 처음부터 끝까지 똑같은 패치 크기($16 \times 16$)를 유지했네. 하지만 객체 탐지 모델(FPN 등)은 얕은 층에서는 디테일(고해상도)을, 깊은 층에서는 숲(저해상도, 큰 수용 영역)을 보아야 하네.

Swin Transformer는 CNN의 풀링(Pooling) 계층을 모방한 패치 병합(Patch Merging)을 통해 이 계층적(Hierarchical) 구조를 완벽하게 부활시켰네.

* 층을 지날 때마다 $2 \times 2$ 크기의 인접한 패치 4개를 하나로 합치네(Concatenation).
* 특징 맵의 해상도(가로세로 크기)는 $\frac{1}{2}$ 로 줄어들고, 정보(채널 수 $C$)는 4배가 되지.
* 여기에 선형 계층(Linear Layer)을 통과시켜 채널을 $2C$ 로 압축하네.
* 결과적으로 ResNet과 똑같이 $H/4 \to H/8 \to H/16 \to H/32$ 형태로 피라미드 구조를 형성하게 되어, 어떤 기존의 객체 탐지 헤드(Faster R-CNN, Mask R-CNN)와도 완벽하게 결합될 수 있게 되었네!

---

## 6. 핵심 용어 및 단축어

* **Swin (Shifted Window):** 창문을 겹치게 이동시켜 지역적(Local) 어텐션의 단절을 극복한 핵심 메커니즘.
* **W-MSA (Window Multi-head Self-Attention):** 전체가 아닌 고정된 윈도우 내에서만 수행하는 어텐션 (연산량 선형화).
* **SW-MSA (Shifted Window Multi-head Self-Attention):** 이전 층 대비 반 칸씩 이동된 윈도우를 사용해 정보의 교류를 만들어내는 어텐션.
* **Cyclic Shift:** 패치들을 반대편으로 밀어 넣어 행렬 연산의 병렬성을 유지하는 효율적인 알고리즘 기법.

---

## 7. PyTorch 코드 예시: 순환 이동의 우아함

Swin Transformer의 핵심인 `Cyclic Shift`가 파이토치에서 얼마나 단순한 함수 하나로 구현되는지 보게. 바로 `torch.roll` 이네!

```python
import torch

# 가로 14, 세로 14 크기의 더미 특징 맵 (배치 1, 채널 C)
# 윈도우 크기(M)를 7이라고 가정해 보겠습니다.
feature_map = torch.arange(14 * 14).view(1, 1, 14, 14)

window_size = 7
shift_size = window_size // 2 # 반 칸 이동이므로 3칸 이동

# torch.roll을 사용하면 위/왼쪽으로 밀려난 픽셀들이 
# 자동으로 아래/오른쪽으로 채워집니다. (이것이 Cyclic Shift!)
shifted_feature_map = torch.roll(
    feature_map, 
    shifts=(-shift_size, -shift_size), # 상하, 좌우로 -3칸씩 밀기
    dims=(2, 3) # 높이(H)와 너비(W) 차원에 적용
)

print("원본 형태:", feature_map.shape)
print("순환 이동 완료. 이제 이 텐서를 고정된 7x7 윈도우로 잘라서 W-MSA에 넣으면 됩니다.")

# 어텐션 연산 후 제자리로 되돌리기 (Reverse Cyclic Shift)
reversed_feature_map = torch.roll(
    shifted_feature_map,
    shifts=(shift_size, shift_size), # 반대 방향으로 다시 +3칸씩 밀기
    dims=(2, 3)
)

# 완벽히 복원됨을 수학적으로 증명
assert torch.equal(feature_map, reversed_feature_map)

```
