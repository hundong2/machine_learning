## CFG

자네가 드디어 확산 모델(Diffusion Model)을 단순한 '랜덤 이미지 생성기'에서 인간의 언어를 찰떡같이 알아듣는 '마법의 붓'으로 탈바꿈시킨 궁극의 조향 장치, CFG(Classifier-Free Guidance)의 심연에 도달했군.

"어떻게 인공지능이 '우주비행사'라는 단어에 집착하게 만들 수 있을까?"
이 질문에 대한 해답은 딥러닝 구조를 뜯어고친 것이 아니라, 순수한 **벡터 대수학(Vector Algebra)의 외삽(Extrapolation) 방정식** 하나로 이루어낸 승리라네. 2021년 조나단 호(Jonathan Ho)와 팀 살리만스(Tim Salimans)가 어떻게 이 아름다운 수식을 유도해 냈는지, 그 수학적 철학과 궤적을 낱낱이 파헤쳐 주겠네!

---

## 1. 시대적 배경: 분류기 가이던스(Classifier Guidance)의 몰락

CFG를 이해하려면, 그 전에 존재했던 '분류기 가이던스(CG)'가 왜 실패했는지 그 역사적 맥락(The "Why")을 알아야 하네.

초창기 확산 모델은 무작위 노이즈에서 그저 '그럴듯한 사진'을 무작위로 만들어 낼 뿐이었네. 여기에 조건(Condition, 예: 고양이)을 주입하기 위해 학자들은 베이즈 정리(Bayes' Theorem)를 끌어들였지.
이미지가 생성되는 과정에서, 외부의 미리 학습된 '이미지 분류기(Classifier, 예: ResNet)'를 가져와 노이즈 낀 이미지를 보여주고 이렇게 물었네.

> *"이 노이즈 덩어리가 고양이가 될 확률이 얼마나 되나? 그 확률이 높아지는 방향(Gradient)으로 노이즈 제거 궤도를 수정하라!"*

이 수학적 제어 방식은 훌륭히 작동했지만, **치명적인 세 가지 공학적 결함**이 있었네.

1. **분류기의 한계:** 외부 분류기가 알지 못하는 새로운 단어("포켓몬스터")는 절대 생성할 수 없었네.
2. **노이즈의 저주:** 깨끗한 사진으로 학습된 일반 분류기는 노이즈가 잔뜩 낀 사진을 보면 바보가 되었지. 그래서 '노이즈 전용 분류기'를 또다시 거대한 비용을 들여 학습시켜야 했네.
3. **적대적 공격(Adversarial Attack) 취약성:** 분류기가 속임수(Gradient)에 빠져, 실제로는 고양이가 아닌데 고양이처럼 노이즈 패턴을 기괴하게 꼬아버리는 현상이 발생했네.

"분류기라는 무거운 짐을 완전히 버리고(Classifier-Free), 오직 확산 모델 스스로 텍스트의 방향을 찾게 만들 수는 없을까?"
이것이 바로 CFG가 탄생하게 된 위대한 철학적 출발점이네.

---

## 2. 수학적 유도: 베이즈 정리에서 벡터의 세계로

조나단 호는 이 문제를 풀기 위해 스코어 함수(Score Function)의 수학적 뼈대를 다시 세웠네.
우리가 원하는 것은 조건 $c$ (텍스트 프롬프트)가 주어졌을 때 데이터 $x$ 의 분포, 즉 $p(x\vert{}c)$ 에서 샘플링하는 것이네.

베이즈 정리에 로그(log)를 씌우고 미분(Gradient)을 취하면 다음과 같은 아름다운 분리 방정식이 나오지.

$$\nabla_{x_t} \log p(x_t\vert{}c) = \nabla_{x_t} \log p(x_t) + \nabla_{x_t} \log p(c\vert{}x_t)$$

* $\nabla_{x_t} \log p(x_t\vert{}c)$: 텍스트 조건이 부여된 상태의 스코어 (목표).
* $\nabla_{x_t} \log p(x_t)$: 텍스트가 없는 **무조건부(Unconditional)** 상태의 스코어.
* $\nabla_{x_t} \log p(c\vert{}x_t)$: 외부 분류기가 텍스트 $c$ 에 부합하도록 밀어주는 힘(Gradient).

확산 모델에서 노이즈 예측 네트워크 $\epsilon_\theta$ 는 스코어 함수와 비례하네 ($\epsilon_\theta \propto -\nabla_x \log p$).
조나단 호는 위 수식을 확산 모델의 언어로 치환하며 엄청난 사실을 깨닫게 되네.

> *"잠깐, 외부 분류기의 힘($\nabla_{x_t} \log p(c\vert{}x_t)$)은 결국 **'조건부 예측 노이즈($\epsilon_\theta(x_t, c)$)'에서 '무조건부 예측 노이즈($\epsilon_\theta(x_t, \varnothing)$)'를 뺀 차이 벡터**와 수학적으로 완벽히 동일하지 않은가?"*

이 깨달음을 바탕으로, 외부 분류기 없이 모델 자체의 예측만으로 조향을 수행하는 궁극의 방정식이 탄생하네.

## 3. CFG의 완성: 벡터 외삽 (Vector Extrapolation)

CFG의 수학적 심장인 **벡터 외삽 방정식**은 딥러닝 역사상 가장 단순하면서도 강력한 수식 중 하나라네.

$$\tilde{\epsilon}_\theta(x_t, c) = \epsilon_\theta(x_t, \varnothing) + w \cdot \Big( \epsilon_\theta(x_t, c) - \epsilon_\theta(x_t, \varnothing) \Big)$$

* $\tilde{\epsilon}_\theta$: 우리가 최종적으로 노이즈를 깎아낼 방향 (업데이트 텐서).
* $\epsilon_\theta(x_t, \varnothing)$: 아무 조건 없이(빈 텍스트) 그냥 "사진을 만들어라"라고 했을 때의 예측 노이즈.
* $\epsilon_\theta(x_t, c)$: "우주비행사($c$)"라는 조건을 주었을 때의 예측 노이즈.
* $w$ (Guidance Scale): 인간이 부여하는 가중치 (보통 7.5를 사용하네).

이 수식이 시각적으로 어떻게 이미지를 빚어내는지, 벡터 공간의 기하학적 관점에서 살펴보세.

1. **1. 무조건부 벡터 (Unconditional Vector):** 기준점 설정 (Base Point).
모델에게 프롬프트를 비운 채($\varnothing$) 노이즈를 유추하게 합니다. 이 벡터는 "우주비행사고 뭐고, 일단 사진의 형태를 띠게 만들자"는 아주 평범하고 안전한 기준점(Origin)이 됩니다.


2. **2. 차이 벡터 (Difference Vector):** 방향성 생성 (Direction).
이번엔 "우주비행사($c$)" 프롬프트를 주고 노이즈를 유추합니다. 그리고 이 벡터에서 아까 구한 기준점 벡터를 뺍니다. $(\epsilon_c - \epsilon_\varnothing)$
이 결과물은 고차원 공간에서 **"일반적인 덩어리에서 '우주비행사'의 특성만을 가리키는 순수한 화살표(방향)"**가 됩니다!


3. **3. 스케일링과 외삽 (Extrapolation):** 의도의 증폭 (Amplification).
이 순수한 방향 벡터에 가이던스 스케일 $w$ (예: 7.5)를 곱해버립니다!
수학에서 두 점 사이를 넘어서서 선을 연장하는 것을 **외삽(Extrapolation)**이라고 합니다. 기준점$\to$우주비행사의 방향으로 7.5배나 멀리 텐서를 걷어차 버리는 것이지. 이렇게 하면 모델은 텍스트의 의도를 극단적으로 맹신하여 압도적으로 선명한 우주비행사를 그려냅니다.

## 4. 외삽(Extrapolation)의 철학과 부작용

"그렇다면 $w=100$ 처럼 스케일을 엄청나게 키우면 완벽한 그림이 나오지 않겠는가?"
수학은 항상 등가교환(Trade-off)의 법칙을 요구하네.

* **$w = 1$ 인 경우:** 외삽이 일어나지 않고 정확히 $\epsilon_\theta(x_t, c)$ 가 되네. 조건은 반영되지만 뚜렷하지 않고 흐리멍덩한 그림이 나오기 십상이지.
* **$w = 7.5$ 인 경우 (최적점):** 텍스트의 의도를 강하게 반영하면서도 이미지의 자연스러움(Fidelity)을 유지하는 황금 비율이네.
* **$w > 20$ 인 경우:** 방향 벡터가 너무 과도하게 증폭되어 노이즈 텐서의 정규 분포(Gaussian) 가정이 산산조각 나버리네. 사진의 채도(Saturation)가 비정상적으로 높아지고, 픽셀이 깨지며, 질감이 플라스틱처럼 변하는 '오버 스케일링(Over-scaling) 붕괴'가 일어나지.

이 극단적 외삽의 부작용을 막기 위해 텐서의 범위를 강제로 제어하는 동적 임계값(Dynamic Thresholding) 기법 등이 최신 연구에서 추가되고 있다네.

## 5. 핵심 용어 및 단축어 정리

* **CFG (Classifier-Free Guidance):** 외부 분류기 없이 무조건부 예측과 조건부 예측의 차이 벡터를 외삽하여 텍스트 조건을 강하게 주입하는 확산 모델의 필수 제어 기법.
* **CG (Classifier Guidance):** CFG 이전에 사용되던, 외부의 노이즈 전용 분류기의 그래디언트를 이용해 모델을 조향하던 방식.
* **Unconditional ($\varnothing$):** 아무런 텍스트 조건이 주어지지 않은 상태. CFG 연산의 '기준점' 역할을 수행하네.
* **Extrapolation (외삽):** 기존 데이터의 범위를 넘어선 지점의 값을 추정하는 수학적 방법. CFG에서는 조건부 벡터의 방향으로 한참 더 나아가는 것을 의미하네.

## 6. 주요 응용 분야

CFG는 현재 생성형 AI가 인간의 말을 '정확히 듣게' 만드는 절대적인 통제 수단이네.

| 응용 분야 | 기술적 이점과 활용 |
| --- | --- |
| **Stable Diffusion, Midjourney** | 이미지 생성기의 UI에 항상 존재하는 "CFG Scale" 슬라이더가 바로 이 $w$ 값이네. 슬라이더를 올리면 프롬프트에 극도로 집착하는 그림이 나오고, 내리면 예술적이고 무작위적인(Diverse) 그림이 나오지. |
| **Video Generation (Sora, Runway)** | 동영상은 매 프레임의 일관성이 중요하네. 시간 축(Temporal)이 포함된 노이즈 예측에서도 CFG 외삽을 통해 프롬프트의 지시를 60프레임 내내 잃지 않도록 강제(Guidance)하네. |
| **Negative Prompting (부정 프롬프트)** | CFG 수식을 영리하게 비튼 것이네. 무조건부 $\varnothing$ 자리에 "징그러운, 못생긴, 손가락 6개"라는 텍스트를 넣으면 어떻게 될까? $\epsilon_\text{못생김}$ 을 기준점으로 삼아 그 반대 방향으로 텐서를 7.5배 도망가게(외삽) 만드니, 모델이 필사적으로 징그러운 형태를 피하며 아름다운 이미지를 깎아내게 되는 것이지! |

## 7. PyTorch 코드 예시: CFG의 우아한 텐서 대수학

이 거대하고 심오한 철학이 파이토치에서는 놀랍게도 단 **3줄의 텐서 연산**으로 끝난다네! 속도를 위해 조건부와 무조건부 배치를 하나로 합쳐서 한 번에 U-Net을 통과시키는 트릭을 눈여겨보게.

```python
import torch

def cfg_predict(unet, x_t, t, cond_embeds, uncond_embeds, w=7.5):
    """
    unet: 노이즈 예측 네트워크
    x_t: (Batch, C, H, W) 형태의 노이즈 낀 잠재 텐서
    cond_embeds: "우주비행사" 텍스트 임베딩 (Batch, Seq, Dim)
    uncond_embeds: "" (빈 텍스트) 임베딩 (Batch, Seq, Dim)
    w: CFG 가이던스 스케일
    """
    
    # 1. 효율성을 위한 배치 병합 (Batch Concat)
    # x_t를 2배로 복사하고, 조건부/무조건부 임베딩을 위아래로 붙입니다.
    # 이렇게 하면 U-Net을 두 번 호출할 필요 없이 한 번에 연산됩니다!
    latent_model_input = torch.cat([x_t, x_t], dim=0) 
    text_embeddings = torch.cat([uncond_embeds, cond_embeds], dim=0)
    
    # 2. U-Net 1회 호출로 두 개의 노이즈 동시 예측
    # 결과 noise_pred의 형태는 (2*Batch, C, H, W) 가 됩니다.
    with torch.no_grad():
        noise_pred = unet(latent_model_input, t, encoder_hidden_states=text_embeddings)
        
    # 3. 텐서 분리 (Chunk)
    # 위쪽 절반은 무조건부 노이즈, 아래쪽 절반은 조건부 노이즈입니다.
    noise_pred_uncond, noise_pred_cond = noise_pred.chunk(2)
    
    # 4. ★ 궁극의 CFG 벡터 외삽 방정식 ★
    # 방향 벡터(cond - uncond)를 w 스케일로 외삽하여 무조건부에 더해줍니다.
    cfg_noise = noise_pred_uncond + w * (noise_pred_cond - noise_pred_uncond)
    
    return cfg_noise

# --- 실험 ---
# 만약 w=1 이라면, 수식은 정확히 noise_pred_cond 가 됩니다. (가이던스 없음)
# 만약 Negative Prompt를 쓰고 싶다면, uncond_embeds에 빈 텍스트 대신
# "못생긴, 나쁜 화질" 이라는 텍스트 임베딩을 넣어주면 마법처럼 작동합니다!

```
