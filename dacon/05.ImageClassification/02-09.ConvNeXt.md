## ConvNeXt(컨브넥스트)

Swin Transformer가 컴퓨터 비전의 모든 왕좌(분류, 탐지, 분할)를 차지했던 2021년, 메타(Meta, 구 Facebook)의 AI 연구소(FAIR) 소속 좡 리우(Zhuang Liu) 연구팀은 아주 도발적이고 본질적인 수학적 질문을 던졌네.

> *"트랜스포머가 CNN을 이긴 것이 정말 **'자아-주의(Self-Attention)'**라는 수학적 연산 방식 때문인가? 아니면 지난 수년간 트랜스포머 진영이 찾아낸 **'거시적/미시적 아키텍처 설계 기법(Macro/Micro Design)'**들이 우월했기 때문일까?"*

이들은 가장 순수하고 고전적인 ResNet-50 모델을 꺼내 들고, Swin Transformer가 사용한 설계 기법들을 하나씩 이식해 보기 시작했네. 그리고 경악스러운 결과를 얻었지. 어텐션 연산 없이 순수 합성곱(CNN)만으로 Swin Transformer의 성능을 박살 내버린 것이네!

이 위대한 반격의 역사와 수학적 설계 철학을 단계별로 해부해 보세.

---

## 1. 공간과 채널의 수학적 분리 (Spatial vs Channel Mixing)

ConvNeXt 설계의 가장 거대한 철학은 "트랜스포머의 어텐션(Attention)이 수학적으로 도대체 무엇을 하고 있는가?"를 CNN의 언어로 번역한 것이네.

트랜스포머의 Multi-Head Self-Attention(MSA)은 공간 정보(어떤 픽셀이 중요한가)를 섞는 작업과, 채널 정보(특징 차원)를 섞는 작업을 철저하게 분리(Decoupling)하여 수행하네.
연구진은 CNN에서도 이 '분리'를 수학적으로 완벽히 모사할 수 있는 무기를 꺼내 들었지. 바로 깊이별 합성곱(Depthwise Convolution)이네!

일반적인 합성곱은 공간과 채널을 한꺼번에 섞어버리며 막대한 파라미터를 생성하네.

* **일반 Conv 연산량:** $K^2 \cdot C_{in} \cdot C_{out} \cdot H \cdot W$

하지만 깊이별 합성곱은 각 채널(Depth)마다 독립적인 2차원 필터(공간 필터)를 적용한 뒤, $1 \times 1$ 합성곱(Pointwise)으로 채널만 따로 섞어주지.

* **깊이별 Conv 연산량:** $K^2 \cdot C \cdot H \cdot W$
* 결과적으로 연산량이 $\frac{1}{C_{out}}$ 로 극적으로 줄어들며, 트랜스포머의 MSA 메커니즘과 수학적으로 거의 동일한 효과를 내게 된다네!

## 2. 현대화의 여정: ResNet에서 ConvNeXt로

FAIR 연구진이 ResNet-50을 ConvNeXt로 뜯어고친 과정을 시각적으로 따라가 보세.

1. **1. 거시적 설계: Patchify Stem과 비율 조정:** 트랜스포머의 16x16 패치 모사.
ResNet은 초반에 $7 \times 7$ 커널과 풀링으로 이미지를 줄이지만, ViT/Swin은 $4 \times 4$ 크기의 겹치지 않는(Non-overlapping) 패치로 이미지를 조각내지. ConvNeXt는 이를 모사하여 `Conv2d(kernel_size=4, stride=4)`라는 'Patchify Stem'을 도입했네. 또한 각 스테이지의 블록 비율을 ResNet의 (3:4:6:3)에서 Swin의 비율인 (1:1:9:1)로 맞추어 깊은 층에 연산량을 집중시켰네.


2. **2. 역 병목 구조 (Inverted Bottleneck):** Swin의 MLP 블록 모사.
ResNet은 차원을 축소했다가 팽창시키는 병목(Bottleneck) 구조를 쓰네. 하지만 트랜스포머의 MLP 블록은 은닉층에서 차원을 4배로 **팽창(Expand)**시켰다가 다시 축소하지. ConvNeXt는 이 수학적 구조를 그대로 가져와 $1 \times 1$ Conv로 차원을 4배 늘린 후 다시 줄이도록 블록의 앞뒤를 뒤집었네.


3. **3. 대형 커널 (Large Kernel Size):** Swin의 7x7 윈도우 모사.
Swin Transformer는 지역 어텐션을 $7 \times 7$ 윈도우 창 안에서 수행하네. 이를 CNN으로 똑같이 구현하기 위해, 깊이별 합성곱(Depthwise Conv)의 커널 크기를 $3 \times 3$ 에서 **$7 \times 7$** 로 과감히 키웠네! 역 병목 구조 덕분에 차원이 축소된 상태에서 깊이별 합성곱을 수행하므로 파라미터 폭발도 막아냈지.


4. **4. 미시적 설계: GELU와 LayerNorm:** 트랜스포머의 우아함 모사.
트랜스포머는 활성화 함수나 정규화(Normalization) 층을 아주 적게 사용하네. ConvNeXt도 이를 따라 블록당 3개이던 ReLU를 오직 1개의 **GELU**로 바꾸고, 배치 정규화(BatchNorm)를 모조리 **층 정규화(LayerNorm)**로 교체했네. 이 미세한 변화들이 모여 최종적인 성능 폭발을 일으켰지.

## 3. 미시적 설계의 수학: GELU와 LayerNorm

초보자들이 흔히 간과하는 것이 바로 이 미시적 수학 함수의 교체라네.

### A. 활성화 함수: ReLU $\to$ GELU

전통적인 CNN은 $f(x) = \max(0, x)$ 인 ReLU를 썼네. 음수는 자비 없이 0으로 죽여버리는 거친 함수지. 반면 자연어 처리에서 넘어온 GELU (Gaussian Error Linear Unit)는 확률론적이고 부드러운(Smooth) 접근을 하네.

$$\text{GELU}(x) = x \Phi(x) = x \cdot \frac{1}{2} \left[ 1 + \text{erf}\left(\frac{x}{\sqrt{2}}\right) \right]$$

* $\Phi(x)$: 표준 정규 분포의 누적 분포 함수(CDF).
* **의미:** 입력값 $x$ 에 "이 값이 정규 분포에서 얼마나 큰 위치에 있는가?"라는 확률값을 곱해주네. 음수여도 완전히 0으로 꺾이지 않고 부드러운 곡선을 그리며, 미분값이 연속적이라 모델이 더 깊고 복잡한 특징 공간을 안정적으로 탐험하게 해주지.

### B. 정규화: BatchNorm $\to$ LayerNorm

배치 정규화(BN)는 미니 배치(Batch) 안의 여러 이미지들을 묶어서 평균과 분산을 구하네. 배치가 작으면 수학적으로 극도로 불안정해지지.
반면 층 정규화(LayerNorm)는 오직 '자기 자신(하나의 이미지)'이 가진 채널들의 픽셀값들만 가지고 평균과 분산을 구하네. 트랜스포머가 시퀀스를 다룰 때 쓰던 이 방식을 CNN에 적용했더니, 학습 안정성이 비약적으로 상승했네!

## 4. 단축어 및 핵심 용어

* **ConvNeXt:** Convolution(합성곱)과 NeXt(다음 세대)의 합성어. 트랜스포머의 장점을 흡수한 차세대 CNN 아키텍처.
* **DWConv (Depthwise Convolution):** 입력 채널들을 서로 섞지 않고, 각 채널마다 독립적인 2차원 공간 필터를 적용하는 합성곱.
* **Inverted Bottleneck (역 병목):** 입력 차원을 줄였다 키우는 기존 방식(모래시계 형태)의 반대로, 먼저 차원을 넓혀 풍부한 특징을 뽑은 뒤 마지막에 압축하는 구조 (항아리 형태).
* **Stem (줄기):** 원본 이미지가 모델에 들어올 때 가장 먼저 만나는 전처리 합성곱 계층. (Patchify Stem은 이미지를 바둑판처럼 썰어내는 역할)

## 5. 파이토치(PyTorch) 코드: ConvNeXt 블록의 우아함

백문이 불여일타! 수많은 설계 철학이 녹아든 ConvNeXt 블록이 실제 코드로 얼마나 우아하고 단순하게 구현되었는지 살펴보게.

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class ConvNeXtBlock(nn.Module):
    def __init__(self, dim, drop_path=0.):
        super().__init__()
        
        # 1. Depthwise Conv (대형 7x7 커널)
        # groups=dim 으로 설정하면 채널이 섞이지 않는 Depthwise 연산이 됩니다.
        self.dwconv = nn.Conv2d(dim, dim, kernel_size=7, padding=3, groups=dim)
        
        # 2. LayerNorm (트랜스포머의 정규화 기법 이식)
        # PyTorch의 LayerNorm은 주로 (B, N, C) 형태를 기대하므로, 
        # (B, C, H, W)를 (B, H, W, C)로 바꾸어 적용하는 트릭을 씁니다.
        self.norm = nn.LayerNorm(dim, eps=1e-6)
        
        # 3. 역 병목 (Inverted Bottleneck) 구조의 Pointwise Conv
        # 채널 차원을 4배로 팽창 (Swin의 MLP 은닉층 모사)
        self.pwconv1 = nn.Linear(dim, 4 * dim) 
        
        # 4. GELU 활성화 함수 (ReLU 대신 사용, 블록당 오직 1개만!)
        self.act = nn.GELU()
        
        # 5. 차원 축소
        # 팽창된 4배의 채널을 다시 원래 차원(dim)으로 복구
        self.pwconv2 = nn.Linear(4 * dim, dim)
        
        # ResNet의 잔차 연결(Residual Connection)에서 사용하는 채널별 스케일링 파라미터
        self.gamma = nn.Parameter(torch.ones((dim)), requires_grad=True)

    def forward(self, x):
        input = x
        
        # 대형 7x7 커널로 공간 정보(Spatial)만 섞기
        x = self.dwconv(x)
        
        # Conv2d 결과 (B, C, H, W) -> LayerNorm/Linear를 위해 (B, H, W, C)로 축 변경
        x = x.permute(0, 2, 3, 1) 
        
        # 정규화 및 트랜스포머 모사 부분
        x = self.norm(x)
        x = self.pwconv1(x)
        x = self.act(x)
        x = self.pwconv2(x)
        x = self.gamma * x
        
        # 다시 원래 차원 순서로 복구: (B, H, W, C) -> (B, C, H, W)
        x = x.permute(0, 3, 1, 2)
        
        # 잔차 연결 (Residual Connection)
        x = input + x
        return x

# --- 실험 ---
# 차원이 96인 56x56 크기의 특징 맵이 들어왔다고 가정합니다.
dummy_input = torch.randn(1, 96, 56, 56)
convnext_block = ConvNeXtBlock(dim=96)

out = convnext_block(dummy_input)
print("ConvNeXt 블록 출력 텐서 크기:", out.shape)
# 출력: torch.Size([1, 96, 56, 56]) -> 크기와 차원이 완벽히 유지됨!

```
