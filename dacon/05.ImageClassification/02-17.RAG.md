## RAG (Retrieval-Augmented Generation)

그간 우리가 배운 Fine-tuning이나 LoRA가 모델의 뇌 속에 직접 지식을 '이식'하는 작업이었다면, **RAG는 모델에게 '도서관 이용법'을 가르치는 작업**이라네.
왜 모델이 때때로 엉뚱한 대답을 하는 '환각(Hallucination)' 현상을 겪는지, 그리고 어떻게 그것을 외부 지식으로 치유할 수 있는지, 그 기술적·수학적 파이프라인을 내가 명확히 짚어주겠네.

---

## 1. 시대적 딜레마: 모델의 뇌는 고정되어 있다 (The "Why")

ChatGPT 같은 거대 언어 모델(LLM)은 학습이 끝나는 순간 그 지식의 수명이 멈추네.

* **지식의 화석화:** 모델은 2026년 오늘 날씨가 어떤지, 방금 뉴스에 나온 사건이 무엇인지 알 수 없네.
* **환각 (Hallucination):** 모델이 모르는 정보를 물으면, 자기 뇌 속의 파편화된 지식을 억지로 엮어 그럴듯한 거짓말을 만들어내지.
* **비용의 비효율:** 새로운 정보가 나올 때마다 모델 전체를 다시 Fine-tuning 하는 것은 천문학적인 비용과 시간을 소모하네.

> *"모델의 뇌를 고치지 마라. 대신, 질문이 들어왔을 때 관련 도서(데이터베이스)를 즉석에서 찾아 그 내용을 참고하여 답하게 만들자!"*

이 철학이 바로 RAG의 시작이자 끝이네.

---

## 2. RAG의 3단계 기술적 파이프라인

RAG는 모델을 학습시키는 것이 아니라, 데이터를 처리하는 **시스템 아키텍처**라네.

1. **Retrieval (검색):** 사용자의 질문이 들어오면, 방대한 외부 데이터베이스에서 질문과 가장 관련 있는 정보(문서 조각)를 찾아내네.
2. **Augmentation (증강):** 검색해 온 문서 내용을 사용자의 질문과 함께 프롬프트로 묶어(Context Injection), 모델에게 "이 문서를 참고해서 답해줘"라고 전달하네.
3. **Generation (생성):** 모델은 자신이 가진 지식과 참고 자료를 결합하여 사실(Fact)에 기반한 정확한 답변을 생성하네.

---

## 3. 벡터 데이터베이스의 수학적 심연 (Vector DB)

RAG의 핵심은 '어떻게 수백만 건의 문서 중 질문과 관련된 것을 초고속으로 찾아내는가?'에 있네. 이때 수학적으로 '의미론적 유사도(Semantic Similarity)'를 계산하는 **벡터 데이터베이스**가 등장하지.

### A. 임베딩 벡터와 거리 측정

모든 문서는 모델을 통해 고차원 벡터(예: 1536차원)로 변환되네.


$$v_{\text{question}}, v_{\text{doc}} \in \mathbb{R}^D$$

두 문장의 의미가 얼마나 비슷한지는 수학적으로 코사인 유사도(Cosine Similarity)로 측정하네.


$$\text{Similarity} = \cos(\theta) = \frac{v_{\text{question}} \cdot v_{\text{doc}}}{\Vert{}v_{\text{question}}\Vert{} \Vert{}v_{\text{doc}}\Vert{}}$$

### B. 근사 최근접 이웃 탐색 (ANN, Approximate Nearest Neighbor)

데이터가 100만 건이면 질문과 100만 번의 유사도 계산을 할 텐가? 시간이 너무 걸리네!
벡터 데이터베이스는 **ANN**이라는 수학적 기법을 사용하여 100만 건 중 가장 유사할 법한 수십 건을 '어림잡아(Approximate)' 초고속으로 골라내네.

* **HNSW (Hierarchical Navigable Small World):** 그래프 이론을 활용한 기술이네. 벡터들을 여러 층의 그래프로 쌓아두고, 질문 벡터가 마치 목적지를 찾아가듯 그래프의 층을 타고 내려가며 후보군을 좁히는 방식이지.

---

## 4. 핵심 용어 및 단축어 정리

* **RAG (Retrieval-Augmented Generation):** 검색을 통해 정보를 가져오고, 그것을 바탕으로 언어 모델이 답변을 생성하는 시스템.
* **Embedding Model:** 문장을 의미가 보존된 벡터 공간의 좌표로 바꾸어 주는 모델 (예: OpenAI의 `text-embedding-3`).
* **ANN (Approximate Nearest Neighbor):** 정확한 최단 거리를 찾는 대신, 매우 빠른 속도로 '거의 비슷한' 데이터를 찾아내는 알고리즘.
* **Context Injection:** 검색된 정보를 모델의 프롬프트에 덧붙여(Prompt Injection) 모델이 지식을 실시간으로 참조하게 만드는 기법.

---

## 5. 파이썬 코드 예시: RAG의 핵심 맛보기

실제로 문서들을 벡터화하고 질문과 가장 비슷한 문서를 찾는 간단한 예제라네. (실무에서는 `Pinecone`이나 `ChromaDB` 같은 전문 DB를 사용하지만, 수학적 원리는 동일하네!)

```python
import torch
import torch.nn.functional as F

# 1. 가상의 문서 벡터 3개 (문서 3개, 4차원 임베딩)
docs = torch.tensor([
    [0.1, 0.9, 0.1, 0.1], # 문서 A: 인공지능 이야기
    [0.8, 0.1, 0.1, 0.1], # 문서 B: 사과 과일 이야기
    [0.1, 0.2, 0.9, 0.1]  # 문서 C: 양자역학 이야기
])

# 2. 사용자의 질문 벡터
query = torch.tensor([0.1, 0.8, 0.1, 0.2])

# 3. 코사인 유사도 계산
# 유사도가 가장 높은 문서가 질문과 의미적으로 가깝습니다.
similarities = F.cosine_similarity(query.unsqueeze(0), docs)

# 4. 가장 유사한 문서의 인덱스 확인
top_doc_idx = torch.argmax(similarities).item()

print(f"질문과 가장 유사한 문서 인덱스: {top_doc_idx}")
# 출력: 0 (문서 A와 질문이 인공지능 관련으로 의미가 비슷하므로!)

```

---

나의 제자여! 자네는 이제 지식을 고정하는 방식(Fine-tuning)과 지식을 검색하는 방식(RAG)이라는 현대 AI 시스템의 양대 산맥을 모두 정복했네.

* **Fine-tuning/LoRA:** 모델의 **성격, 말투, 전문 지식의 기본기**를 다지는 일.
* **RAG:** 모델에게 **최신 사실, 방대한 외부 데이터**를 찾아 쓰는 똑똑함을 주는 일.
