# LoRA(Low-Rank Adaptation)

나의 열정적인 제자여! 딥러닝의 거대한 파라미터라는 괴물을, 마치 마법처럼 **'행렬 분해(Matrix Decomposition)'라는 우아한 수학적 메스(Scalpel)로 다루어내는 예술의 경지**, 그 정점에 선 LoRA(Low-Rank Adaptation)의 세계에 온 것을 환영하네.

자네는 이제 모델의 뇌 전체를 바꾸는 것이 아니라, 뇌의 특정 부위에 아주 얇은 신경 다발을 이식하는 법을 배우게 될 것이네. 이것이 바로 현대 생성형 AI(Stable Diffusion, LLM 등)가 개인의 취향을 학습하는 비밀이지.

---

## 1. 시대적 딜레마: 모델이 너무 거대해졌다 (The "Why")

수십억, 수천억 개의 파라미터를 가진 거대 모델을 우리 개인의 컴퓨터에서 학습(Fine-tuning)시키는 것은 물리적으로 불가능하네.
기존의 Full Fine-tuning(전체 튜닝)은 모델의 모든 가중치($W \in \mathbb{R}^{d \times k}$)를 업데이트해야 하네. 이는 엄청난 VRAM을 소모할 뿐만 아니라, 모델이 원래 가지고 있던 지식을 잊어버리는 **파괴적 망각(Catastrophic Forgetting)** 문제까지 안고 있었지.

2021년, 마이크로소프트 연구진은 여기서 수학적인 통찰을 던지네.

> *"과연 우리는 모델의 모든 가중치를 다 바꿔야 할까? 학습 중에 발생하는 가중치의 변화량($\Delta W$)은, 사실 매우 낮은 **'내재적 차원(Intrinsic Dimension)'**을 가지고 있지 않을까?"*

이 가설이 바로 '가중치 변화의 저랭크(Low-Rank) 가설'이네.

---

## 2. 수학적 원리: 행렬의 랭크(Rank) 분해

전체 가중치 행렬 $W_0$ 가 $d \times k$ 차원이라고 할 때, 우리가 학습을 통해 얻고자 하는 업데이트 행렬 $\Delta W$ 를 $W_0 + \Delta W$ 라고 정의하세.
LoRA의 핵심은 **$\Delta W$를 직접 학습하는 대신, 두 개의 훨씬 작은 행렬의 곱으로 분해(Decompose)하여 학습**하는 것이네.

### 수식적 정의

$$W = W_0 + \Delta W = W_0 + BA$$

* $W_0$: 고정된(Frozen) 원본 가중치. (학습 절대 금지!)
* $A \in \mathbb{R}^{r \times d}$: $r$은 우리가 정하는 **랭크(Rank)**. ($r \ll \min(d, k)$)
* $B \in \mathbb{R}^{k \times r}$: $r$은 위와 동일.

**[왜 랭크 분해인가?]**
만약 $W_0$ 가 $1000 \times 1000$ 행렬(파라미터 100만 개)이라면, $\Delta W$도 100만 개를 학습해야 하네.
하지만 랭크 $r=8$ 로 분해하면:

* $A$: $8 \times 1000$ (8,000개)
* $B$: $1000 \times 8$ (8,000개)
* 총 파라미터: **16,000개!**

약 **62배**나 적은 파라미터로 거의 동일한 성능의 업데이트를 수행할 수 있게 되는 것이네. 이것이 바로 선형대수학의 마법이지.

---

## 3. 학습의 과정과 추론 시의 수학적 결합

LoRA가 경이로운 이유는 **학습 단계와 추론 단계의 수학적 대칭성** 때문이네.

* **학습 시:** $W_0$ 는 얼려두고 $A$와 $B$만 업데이트하네. 원본 $W_0$를 통과한 값과 $BA$를 통과한 값을 합산하여 최종 출력($Y$)을 생성하지.

$$Y = W_0x + BAx = (W_0 + BA)x$$


* **추론 시 (Inference):** 여기서 더 놀라운 일이 벌어지네. 우리는 학습이 끝난 후 $BA$를 계산하여 미리 구한 $W' = W_0 + BA$ 를 만들 수 있네.
**즉, 추론 단계에서는 LoRA를 전혀 사용하지 않은 원본 모델과 속도가 100% 동일하네!** (추가 연산량이 0)

이것이 다른 튜닝 기법(Adapter 등)과의 결정적인 차이네. 다른 기법들은 모델의 층 사이에 새로운 레이어를 끼워 넣어 추론 속도를 늦추지만, LoRA는 원본 모델에 그대로 흡수되어 사라지기 때문이지.

---

## 4. 단축어 및 핵심 용어 정리

* **LoRA (Low-Rank Adaptation):** 거대 행렬의 업데이트 행렬을 저랭크 행렬의 곱으로 분해하여 학습 파라미터를 획기적으로 줄이는 기법.
* **Rank ($r$):** 행렬 분해에서 사용하는 차원의 크기. 보통 4, 8, 16 등 작은 수를 사용하며, $r$이 작을수록 모델은 더 추상적인 스타일 위주로 학습하게 됨.
* **Frozen Weight:** 학습 과정에서 가중치가 갱신되지 않도록 잠가둔(Lock) 고정된 가중치.
* **Intrinsic Dimension:** 복잡한 시스템 내부에서 실제 데이터의 변화를 설명하는 데 필요한 최소한의 차원. 인공지능이 실제로는 수천억 개의 파라미터가 없어도, 핵심 스타일은 몇만 개만으로도 충분히 학습 가능하다는 수학적 가정.

## 5. 주요 응용 분야

| 분야 | 활용 방식 |
| --- | --- |
| **개인화 이미지 생성 (Stable Diffusion)** | 사용자의 얼굴 사진 10~20장을 LoRA로 학습시키면, 그 모델은 5MB 정도의 작은 파일이 되네. 이를 불러오기만 하면 본인의 얼굴을 모델에 즉시 이식할 수 있지. |
| **특정 화풍(Style) 주입** | 수채화, 사이버펑크, 혹은 특정 작가의 화풍을 학습한 LoRA 파일들을 배포하여, 누구나 자신의 생성 AI에 해당 화풍을 씌울 수 있게 되었네. |
| **언어 모델 튜닝 (LLM)** | ChatGPT(Llama 3 등)와 같은 거대 모델을 특정 산업(의료, 법률)용으로 튜닝할 때, 전체 모델을 복제하지 않고 작은 LoRA 파일 하나만 배포하면 되므로 배포 효율이 압도적임. |

## 6. 파이토치(PyTorch) 코드 예시: LoRA의 수학적 구현

실제 구현은 아주 단순하네. 기존 선형 레이어(Linear)를 대체하는 `LoRALinear` 클래스를 보게나.

```python
import torch
import torch.nn as nn
import math

class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, rank=8):
        super().__init__()
        # 1. 원본 가중치 (고정됨)
        self.linear = nn.Linear(in_features, out_features)
        self.linear.weight.requires_grad = False # 학습 금지!
        
        # 2. 저랭크 행렬 A, B 정의
        self.lora_A = nn.Parameter(torch.zeros(rank, in_features))
        self.lora_B = nn.Parameter(torch.zeros(out_features, rank))
        
        # 가중치 초기화 (A는 가우시안, B는 0으로 시작하여 학습 초기에 오차가 없음)
        nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))
        nn.init.zeros_(self.lora_B)

    def forward(self, x):
        # 3. Y = Wx + BAx 연산
        original_output = self.linear(x)
        lora_output = (x @ self.lora_A.T @ self.lora_B.T)
        return original_output + lora_output

# --- 실험 ---
# 1024차원 입력을 받아 1024차원으로 출력하는 거대 행렬 층
layer = LoRALinear(1024, 1024, rank=8)
input_tensor = torch.randn(1, 1024)

# 4. 연산 확인
output = layer(input_tensor)
print("LoRA 적용 결과 형태:", output.shape)
# 학습 시: self.lora_A, self.lora_B만 업데이트되므로 매우 빠름!

```
