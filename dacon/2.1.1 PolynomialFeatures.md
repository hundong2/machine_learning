# PolynomialFeatures 설명
PolynomialFeatures는 기존 입력 변수들을 조합해서 제곱항, 세제곱항, 변수 간 곱 같은 새로운 피처를 자동으로 만들어주는 전처리 도구입니다. 선형 모델이 직접 표현하기 어려운 비선형 관계를 더 잘 학습하도록 도와줄 때 자주 사용합니다.

## 예시 파일
[scikit-learn PolynomialFeatures 공식 예시](https://scikit-learn.org/stable/modules/generated/sklearn.preprocessing.PolynomialFeatures.html)

## 답변
PolynomialFeatures의 핵심은 원래 피처를 더 풍부하게 확장하는 것입니다.

예를 들어 원래 데이터가 아래처럼 두 변수라고 하겠습니다.

    x1, x2

여기서 degree=2를 적용하면 보통 아래 피처들이 생성됩니다.

    1, x1, x2, x1^2, x1*x2, x2^2

즉, 단순히 원래 변수만 쓰는 것이 아니라
x1의 제곱,
x2의 제곱,
x1과 x2의 상호작용 항까지 같이 모델에 넣게 됩니다.

이렇게 하면 선형 회귀 같은 모델도 더 복잡한 곡선 형태를 어느 정도 표현할 수 있습니다.

가장 기본적인 사용 예시는 아래와 같습니다.

    from sklearn.preprocessing import PolynomialFeatures
    import numpy as np

    X = np.array([[2, 3],
                  [4, 5],
                  [6, 7]])

    poly = PolynomialFeatures(degree=2)
    X_poly = poly.fit_transform(X)

    print(X_poly)

출력은 대략 이런 구조입니다.

    [[ 1.  2.  3.  4.  6.  9.]
     [ 1.  4.  5. 16. 20. 25.]
     [ 1.  6.  7. 36. 42. 49.]]

각 열의 의미는 순서대로 다음과 같습니다.

1. 상수항 1
2. x1
3. x2
4. x1^2
5. x1*x2
6. x2^2

실제로 어떤 피처가 생성됐는지는 아래처럼 확인할 수 있습니다.

    print(poly.get_feature_names_out(["x1", "x2"]))

결과는 보통 아래처럼 나옵니다.

    ['1' 'x1' 'x2' 'x1^2' 'x1 x2' 'x2^2']

자주 쓰는 주요 옵션은 아래 3개입니다.

1. degree
몇 차항까지 만들지 정합니다.
degree=2면 제곱항까지, degree=3이면 세제곱항까지 생성합니다.

2. include_bias
맨 앞의 상수항 1을 포함할지 정합니다.
기본값은 True입니다.
선형 회귀 모델이 이미 절편을 따로 학습한다면 False로 두는 경우도 많습니다.

예시:

    poly = PolynomialFeatures(degree=2, include_bias=False)

그러면 결과는

    x1, x2, x1^2, x1*x2, x2^2

처럼 시작합니다.

3. interaction_only
변수의 자기 자신 제곱은 빼고, 변수끼리 곱한 항만 만들고 싶을 때 사용합니다.

예시:

    poly = PolynomialFeatures(degree=2, interaction_only=True, include_bias=False)

그러면 생성되는 피처는 대략 아래처럼 됩니다.

    x1, x2, x1*x2

즉 x1^2, x2^2는 제외됩니다.

언제 쓰면 좋냐도 중요합니다.

1. 선형 회귀가 너무 단순해서 곡선 형태를 못 잡을 때
2. 변수끼리의 상호작용이 중요할 때
3. 트리 모델이 아니라 선형 계열 모델을 유지하면서 표현력을 높이고 싶을 때

예를 들어
면적이 커질수록 가격이 무조건 직선적으로 오르지 않고,
어느 구간부터 더 가파르게 증가하는 경우가 있습니다.
이럴 때 제곱항이 도움이 될 수 있습니다.

실무에서는 보통 모델과 함께 파이프라인으로 묶어서 사용합니다.

    from sklearn.pipeline import make_pipeline
    from sklearn.preprocessing import PolynomialFeatures, StandardScaler
    from sklearn.linear_model import LinearRegression

    model = make_pipeline(
        PolynomialFeatures(degree=2, include_bias=False),
        StandardScaler(),
        LinearRegression()
    )

이렇게 쓰는 이유는 다항식 피처를 만들면 값의 크기가 급격히 커질 수 있어서, 스케일링이 같이 필요한 경우가 많기 때문입니다.

중요한 주의점도 있습니다.

1. 피처 수가 매우 빠르게 늘어납니다.
원래 변수 수가 많을수록 생성되는 컬럼 수가 급격히 증가합니다.

2. 과적합 위험이 커집니다.
degree를 너무 크게 주면 학습 데이터에는 잘 맞지만 새 데이터에서는 성능이 떨어질 수 있습니다.

3. 해석이 어려워질 수 있습니다.
x1^3, x1*x2 같은 항이 많아지면 모델 해석이 복잡해집니다.

4. 모든 모델에 꼭 필요한 것은 아닙니다.
랜덤포레스트, XGBoost 같은 트리 기반 모델은 원래 비선형성을 잘 다루므로 굳이 PolynomialFeatures가 필요 없는 경우가 많습니다.

초보자 관점에서는 이렇게 이해하면 가장 쉽습니다.

1. 원래 컬럼을 더 많은 컬럼으로 확장해주는 도구
2. 제곱, 세제곱, 변수 간 곱을 자동 생성
3. 선형 모델이 비선형 패턴을 배우게 도와줌
4. 하지만 차수를 너무 높이면 과적합과 차원 폭증이 생김

현재 보고 있는 피처 엔지니어링 맥락에서는
온도, 습도, 풍속 같은 변수들이 서로 함께 영향을 줄 수 있으므로
단순 원본 피처만 쓰는 것보다 상호작용 항이 성능 개선에 도움이 될 수 있습니다.
예를 들면 아래와 같은 조합입니다.

1. T * RH
2. WS * RH
3. T^2
4. DNI * DHI

이런 항들을 자동으로 만들어 주는 것이 PolynomialFeatures의 역할입니다.
