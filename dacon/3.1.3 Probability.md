# 3.1.3 Probability

## 1) 확률분포 기초 한눈에 보기

| 구분 | 분포 이름 | 무엇을 예측하는가? (핵심 용도) | 확률 함수 | 그래프 형태 |
|---|---|---|---|---|
| 이산(셀 수 있음) | 베르누이 분포 | 성공/실패, 딱 1번 시도 결과 | PMF(확률질량함수) | 막대 2개 |
| 이산(셀 수 있음) | 이항 분포 | 성공/실패를 n번 반복했을 때 성공 횟수 | PMF | 막대 |
| 이산(셀 수 있음) | 푸아송 분포 | 정해진 시간/공간에서 사건 발생 횟수 | PMF | 오른쪽 꼬리 막대 |
| 연속(이어져 있음) | 균등 분포 | 모든 구간이 같은 가능성일 때 값 | PDF(확률밀도함수) | 직사각형 |
| 연속(이어져 있음) | 정규 분포 | 평균 근처에 값이 몰리는 연속값 | PDF | 종 모양 곡선 |
| 연속(이어져 있음) | 지수 분포 | 사건과 다음 사건 사이의 대기 시간 | PDF | 감소 곡선 |

핵심 구분:
- PMF: 특정 값 하나의 확률을 직접 다룸(이산)
- PDF: 특정 점의 확률이 아니라 구간 넓이로 확률을 해석(연속)

## 2) 분포별 원리 + ML 사용 예시

| 분포 | 기초 원리(왜 이런 모양인가) | 주요 파라미터 | 평균/분산 | ML에서 자주 쓰는 곳 | 실무 예시 |
|---|---|---|---|---|---|
| 베르누이 | 한 번의 시행이 성공(1) 또는 실패(0)만 가짐 | p(성공확률) | 평균: p, 분산: p(1-p) | 이진 라벨 모델링, 로지스틱 회귀의 관측값 가정 | 이메일이 스팸인지(1/0) |
| 이항 | 베르누이 시행을 독립적으로 n번 반복한 성공 횟수 | n, p | 평균: np, 분산: np(1-p) | CTR 예측의 클릭 수 모델링, A/B 테스트 | 100명 노출 중 클릭 수 |
| 푸아송 | 단위 시간/공간에서 드문 사건이 독립적으로 발생 | λ(단위당 평균 발생 수) | 평균: λ, 분산: λ | 카운트 데이터 회귀(Poisson Regression), 이벤트 빈도 예측 | 시간당 콜센터 문의 건수 |
| 균등 | 특정 구간 [a,b]에서 모든 값이 같은 밀도 | a, b | 평균: (a+b)/2, 분산: (b-a)^2/12 | 난수 초기화, 탐색 샘플링 | 하이퍼파라미터 랜덤서치 샘플링 |
| 정규 | 많은 작은 요인의 합이 중심으로 모이는 성질(중심극한정리) | μ, σ^2 | 평균: μ, 분산: σ^2 | 오차항 가정, 가우시안 나이브베이즈, 정규화 후 분석 | 시험 점수/센서 노이즈 모델링 |
| 지수 | 사건 간 대기시간을 설명, 무기억성(memoryless) 특징 | λ(발생률) | 평균: 1/λ, 분산: 1/λ^2 | 생존분석 기초, 도착 간격 모델링 | 서버 요청 간 간격 시간 |

## 3) 어떤 분포를 선택할까? (빠른 의사결정)

| 데이터/문제 형태 | 추천 분포 | 이유 |
|---|---|---|
| 결과가 0 또는 1 | 베르누이 | 이진 결과를 직접 모델링 |
| 0/1 결과의 합계(성공 횟수) | 이항 | 반복 시행의 성공 횟수 구조와 일치 |
| 단위 구간당 사건 발생 건수 | 푸아송 | 카운트 데이터의 대표 가정 |
| 값이 일정 범위에서 균일하게 나옴 | 균등 | 모든 구간 가능성이 동일 |
| 연속값이 평균 주변에 몰림 | 정규 | 자연/측정 데이터에서 가장 흔한 형태 |
| 사건 사이 시간 간격 | 지수 | 대기시간 + 무기억성 설명 가능 |

## 4) 자주 헷갈리는 포인트

| 헷갈림 | 올바른 이해 |
|---|---|
| PDF 값이 1보다 크면 확률이 1보다 큰 것 아닌가? | 아님. 연속분포 확률은 구간 적분값으로 계산 |
| 푸아송과 지수는 별개인가? | 연결됨. 푸아송은 횟수, 지수는 사건 간 시간 |
| 정규분포를 무조건 가정해도 되나? | 아님. 왜도/이상치가 크면 로그변환, 다른 분포 고려 |

## 5) ML 관점 핵심 요약

- 분포 선택은 모델 성능뿐 아니라 해석 가능성과 불확실성 추정 품질을 좌우한다.
- 타깃이 카운트면 푸아송 계열, 이진이면 베르누이/이항 계열이 기본 출발점이다.
- 연속 타깃 회귀에서 오차가 정규라는 가정은 강력하지만, 데이터 진단(잔차 확인)이 먼저다.
