# 성능 평가 지표 쉽게 이해하기
정확도, 정밀도, 재현율, F1 score는 모델이 얼마나 잘 맞추는지를 서로 다른 관점에서 보는 방법입니다. 핵심은 단순히 "몇 개 맞췄나"만 보면 부족하고, "틀린 방식"까지 봐야 한다는 점입니다.

## 예시 파일
[Scikit-learn classification metrics 예시](https://scikit-learn.org/stable/modules/model_evaluation.html#classification-metrics)

가장 먼저 알아야 할 것은 혼동행렬입니다.  
이 4개 지표는 거의 모두 혼동행렬에서 시작합니다.

예를 들어 이진 분류에서 결과를 아래처럼 나눕니다.

기준을 이렇게 잡는 것입니다.

- Positive: 스팸 메일
- Negative: 정상 메일

- 실제 정상 = 실제 Negative = 실제 스팸 아님
- 실제 스팸 = 실제 Positive

- 이 기준으로 다시 보면:

- TP: 실제 스팸, 예측도 스팸
- TN: 실제 정상, 예측도 정상
- FP: 실제 정상인데 스팸이라고 예측
- FN: 실제 스팸인데 정상이라고 예측


1. TP, True Positive  
실제로 양성이고, 모델도 양성이라고 맞춘 경우

1. TN, True Negative  
실제로 음성이고, 모델도 음성이라고 맞춘 경우

1. FP, False Positive  
실제로 음성인데, 모델이 양성이라고 틀리게 예측한 경우

1. FN, False Negative  
실제로 양성인데, 모델이 음성이라고 틀리게 예측한 경우

직관적으로 보면:

- TP: 잘 맞춘 좋은 양성 예측
- TN: 잘 맞춘 좋은 음성 예측
- FP: 괜히 양성이라고 오판
- FN: 놓쳐버린 양성

예를 들어 스팸 메일 분류를 생각하면 이해가 쉽습니다.

- TP: 스팸 메일을 스팸이라고 맞춤
- TN: 정상 메일을 정상이라고 맞춤
- FP: 정상 메일을 스팸이라고 잘못 판단
- FN: 스팸 메일을 정상이라고 놓침

이제 각 지표를 보면 됩니다.

## 정확도
정확도는 전체 중에서 맞춘 비율입니다.

$$
\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
$$

즉,

- 맞춘 개수 / 전체 개수

입니다.

예를 들어 100개 중 90개를 맞췄으면 정확도는 0.9, 즉 90%입니다.

### 정확도의 직관
정확도는 가장 직관적입니다.  
"전체적으로 얼마나 많이 맞췄는가?"를 봅니다.

### 정확도의 문제점
데이터가 불균형하면 정확도는 쉽게 속을 수 있습니다.

예를 들어:

- 전체 1000명
- 실제 양성 10명
- 실제 음성 990명

모델이 무조건 음성이라고만 예측하면:

- 990명 맞춤
- 정확도 99%

겉보기엔 좋아 보이지만, 실제로 중요한 양성 10명은 전부 놓친 것입니다.  
그래서 정확도만 보면 위험합니다.

## 정밀도
정밀도는 모델이 양성이라고 예측한 것들 중에서, 실제로 진짜 양성인 비율입니다.

$$
\text{Precision} = \frac{TP}{TP + FP}
$$

즉,

- 양성이라고 예측한 것 중 얼마나 정확했는가

입니다.

### 정밀도의 직관
정밀도는 "내가 양성이라고 말한 것들을 얼마나 믿을 수 있나?"를 보는 지표입니다.

예를 들어 병원 검사에서 양성 판정을 내릴 때, 실제로 양성인 사람만 잘 골라내고 싶다면 정밀도가 중요합니다.

### 정밀도가 중요한 상황
1. 잘못된 양성 판정이 치명적일 때
2. FP를 줄이고 싶을 때
3. 양성이라고 말할 때 신중해야 할 때

예시:

- 스팸이 아닌 메일을 스팸이라고 보내버리면 문제
- 정상 고객을 사기 고객이라고 분류하면 문제
- 건강한 사람을 중증 질환으로 잘못 분류하면 문제

## 재현율
재현율은 실제 양성들 중에서 모델이 양성이라고 잘 찾아낸 비율입니다.

$$
\text{Recall} = \frac{TP}{TP + FN}
$$

즉,

- 실제 양성 중에서 얼마나 많이 잡아냈는가

입니다.

### 재현율의 직관
재현율은 "놓치지 않는 능력"입니다.

예를 들어 암 진단에서 실제 환자를 최대한 놓치지 않는 것이 중요하다면 재현율이 중요합니다.

### 재현율이 중요한 상황
1. FN을 줄이고 싶을 때
2. 실제 양성을 놓치면 안 될 때
3. 탐지 누락이 치명적일 때

예시:

- 암 환자를 정상으로 놓치는 경우
- 사기 거래를 정상 거래로 놓치는 경우
- 화재 경보 상황을 놓치는 경우

## 정밀도와 재현율의 차이
이 둘은 비슷해 보이지만 보는 방향이 다릅니다.

정밀도는 예측 기준입니다.

- 양성이라고 예측한 것 중 진짜가 얼마나 많나?

재현율은 실제 정답 기준입니다.

- 실제 양성 중 내가 얼마나 많이 찾았나?

쉽게 말하면:

- 정밀도: 맞다고 말한 것의 신뢰도
- 재현율: 찾아내는 능력

## F1 score
F1 score는 정밀도와 재현율을 함께 고려한 지표입니다.

공식은 아래와 같습니다.

$$
F1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}
$$

이것은 정밀도와 재현율의 조화평균입니다.

### 왜 그냥 평균이 아니라 조화평균인가
일반 평균은 한쪽이 매우 낮아도 다른 한쪽이 높으면 적당히 커질 수 있습니다.  
하지만 조화평균은 둘 중 하나가 낮으면 전체 점수를 크게 낮춥니다.

즉, F1 score는

- 정밀도도 좋아야 하고
- 재현율도 좋아야 한다

를 강하게 요구합니다.

### F1의 직관
F1 score는 "양성 예측을 잘하고, 동시에 놓치지도 않는가?"를 균형 있게 보는 점수입니다.

## 숫자로 바로 이해하는 예시
다음 상황을 가정하겠습니다.

- 실제 양성 40명
- 실제 음성 60명

모델 결과가 아래와 같다고 하겠습니다.

- TP = 30
- TN = 50
- FP = 10
- FN = 10

이제 계산해보면:

### 정확도
$$
\text{Accuracy} = \frac{30 + 50}{30 + 50 + 10 + 10} = \frac{80}{100} = 0.8
$$

즉 80%입니다.

### 정밀도
$$
\text{Precision} = \frac{30}{30 + 10} = \frac{30}{40} = 0.75
$$

즉, 양성이라고 예측한 것 중 75%가 진짜 양성입니다.

### 재현율
$$
\text{Recall} = \frac{30}{30 + 10} = \frac{30}{40} = 0.75
$$

즉, 실제 양성 중 75%를 찾아냈습니다.

### F1 score
$$
F1 = 2 \cdot \frac{0.75 \cdot 0.75}{0.75 + 0.75} = 0.75
$$

이 예시에서는 정밀도와 재현율이 같아서 F1도 0.75가 됩니다.

## 정밀도와 재현율이 충돌하는 이유
보통 정밀도와 재현율은 동시에 완벽하게 높이기 어렵습니다.

예를 들어 모델이 양성 판정을 아주 엄격하게 하면:

- 양성이라고 말하는 횟수가 줄어듦
- FP가 줄어들 수 있음
- 정밀도는 올라갈 수 있음
- 하지만 실제 양성을 놓쳐서 FN이 늘 수 있음
- 재현율은 떨어질 수 있음

반대로 모델이 양성 판정을 아주 쉽게 하면:

- 더 많은 양성을 잡음
- FN이 줄 수 있음
- 재현율은 올라갈 수 있음
- 하지만 FP가 늘 수 있음
- 정밀도는 떨어질 수 있음

즉:

- 정밀도를 높이면 재현율이 내려갈 수 있고
- 재현율을 높이면 정밀도가 내려갈 수 있습니다

그래서 F1 score가 균형 평가에 많이 쓰입니다.

## 실제로 어떻게 계산되는가
scikit-learn 같은 라이브러리는 대략 아래 순서로 계산합니다.

1. 정답값과 예측값을 한 개씩 비교
2. TP, TN, FP, FN 개수를 셈
3. 위 개수를 공식에 넣음
4. 정확도, 정밀도, 재현율, F1 score를 계산

즉 특별히 복잡한 마법 같은 알고리즘이 아니라, 결국은 예측 결과를 분류해서 개수를 세고 공식을 적용하는 과정입니다.

## 다중분류에서는 어떻게 되나
현재 코드에서 average='macro'를 쓰고 있으므로 이 부분이 중요합니다.

이진 분류가 아니라 클래스가 여러 개면, 각 클래스를 한 번씩 양성으로 보고 지표를 계산한 뒤 평균을 냅니다.

예를 들어 클래스가 0, 1, 2 세 개라면:

1. 클래스 0을 양성으로 보고 precision, recall, F1 계산
2. 클래스 1을 양성으로 보고 precision, recall, F1 계산
3. 클래스 2를 양성으로 보고 precision, recall, F1 계산
4. 세 값을 평균냄

이것이 macro average입니다.

### macro average의 의미
각 클래스를 똑같이 중요하게 취급합니다.  
샘플 수가 적은 클래스도 동일한 비중으로 반영됩니다.

즉:

- 큰 클래스만 잘 맞춰도 점수가 높아지는 것을 막고
- 작은 클래스의 성능도 중요하게 봅니다

수식으로 쓰면, 클래스가 $K$개일 때:

$$
\text{Macro Precision} = \frac{1}{K}\sum_{k=1}^{K}\text{Precision}_k
$$

$$
\text{Macro Recall} = \frac{1}{K}\sum_{k=1}^{K}\text{Recall}_k
$$

$$
\text{Macro F1} = \frac{1}{K}\sum_{k=1}^{K}\text{F1}_k
$$

## 언제 어떤 지표를 더 봐야 하나
1. 전체적으로 몇 개 맞췄는지 빠르게 보고 싶다  
정확도

2. 양성이라고 예측한 것의 신뢰성이 중요하다  
정밀도

3. 실제 양성을 놓치지 않는 것이 중요하다  
재현율

4. 정밀도와 재현율을 균형 있게 보고 싶다  
F1 score

## 초보자용 한 번에 이해하는 비유
시험 채점으로 비유하면:

- 정확도: 전체 문제 중 몇 개 맞췄나
- 정밀도: 내가 정답이라고 체크한 것 중 실제 정답 비율
- 재현율: 실제 정답 문제를 얼마나 빠짐없이 맞췄나
- F1 score: 정밀도와 재현율을 함께 잘했나

스팸 필터 비유로 보면:

- 정확도: 전체 메일 중 맞게 분류한 비율
- 정밀도: 스팸이라고 분류한 메일 중 진짜 스팸 비율
- 재현율: 실제 스팸 메일 중 잡아낸 비율
- F1 score: 스팸을 잘 잡으면서 정상 메일도 너무 많이 오판하지 않았는지의 균형

## 가장 중요한 핵심 정리
1. 정확도는 전체 맞춘 비율이지만, 불균형 데이터에서는 속을 수 있습니다.
2. 정밀도는 양성 예측의 신뢰도를 봅니다.
3. 재현율은 실제 양성을 얼마나 놓치지 않았는지 봅니다.
4. F1 score는 정밀도와 재현율의 균형을 봅니다.
5. 다중분류에서 average='macro'는 각 클래스를 동일 비중으로 평균내는 방식입니다.
