## 선수 지식과 기호

Part 1의 `xyxy`, `cxcywh`, IoU, NMS를 알고 있다고 가정한다.

| 기호 | 뜻 |
| --- | --- |
| $N$ | batch 크기 |
| $H_k,W_k$ | $k$번째 feature map의 높이와 너비 |
| $D_k$ | 해당 feature map의 stride |
| $A_k$ | 셀마다 배치하는 default box 개수 |
| $C$ | foreground class 개수 |
| $s_k$ | 입력 크기에 대해 정규화한 default box scale |
| $a$ | aspect ratio, $w/h$ |
| $d=(d_x,d_y,d_w,d_h)$ | default box의 `cxcywh` |
| $g=(g_x,g_y,g_w,g_h)$ | 정답 box의 `cxcywh` |
| $t=(t_x,t_y,t_w,t_h)$ | box regression target |
| $C_k$ | backbone의 $k$번째 bottom-up feature |
| $M_k$ | FPN의 lateral과 top-down을 더한 중간 feature |
| $P_k$ | smoothing convolution 뒤의 FPN 출력 |

모든 box 좌표는 별도 언급이 없으면 입력 너비와 높이로 나눈 $[0,1]$ 정규화 좌표다. 이미지 tensor는 PyTorch의 `NCHW`, box는 마지막 축 길이 4인 `cxcywh` 또는 `xyxy`다.

## 1. 직관: 확대경 하나가 아니라 여러 눈금의 지도

YOLOv1의 최종 $7\times7$ 격자만 보면 작은 객체의 중심들이 같은 셀에 들어가기 쉽다. 하지만 문제를 단순히 “한 셀이 입력의 정확한 $64\times64$ 픽셀을 대표한다”라고 이해하면 안 된다. convolution의 receptive field는 이웃 셀끼리 겹치며, 최종 feature 하나가 보는 입력 범위는 단순한 stride보다 클 수 있다.

작은 객체가 어려운 이유는 세 가지가 함께 작동하기 때문이다.

- 깊은 feature map의 큰 stride 때문에 객체 중심과 경계가 거칠게 양자화된다.
- downsampling 과정에서 몇 픽셀짜리 신호가 배경과 섞이거나 사라진다.
- 한 위치가 표현할 수 있는 class·box 슬롯보다 객체가 많으면 target collision이 생긴다.

SSD와 FPN은 둘 다 “여러 해상도”를 쓰지만 역할이 다르다.

| 구조 | 핵심 질문 | 방법 |
| --- | --- | --- |
| SSD | 어느 해상도에서 바로 예측할까? | backbone과 추가 layer의 여러 feature map마다 prediction head를 붙인다. |
| FPN | 고해상도 feature에 깊은 의미를 어떻게 보낼까? | top-down upsample과 lateral connection으로 feature 자체를 융합한다. |

SSD는 multi-scale prediction이고 FPN은 multi-scale representation이다. 둘은 경쟁 개념이 아니며, FPN 위에 detection head를 각 level마다 붙일 수 있다.

## 2. 원본에서 바로잡을 점

원본의 비유는 흐름을 잡는 데 유용하지만 구현 계약으로 옮길 때 다음을 구분해야 한다.

| 원본의 축약 | 수정·보완 |
| --- | --- |
| $448/7=64$이므로 feature 한 셀이 입력의 $64\times64$ 구획을 대표 | 64는 출력 stride에 가까운 공간 간격이다. 실제 receptive field는 더 크고 이웃 위치와 겹친다. 작은 객체 문제는 stride, 정보 손실, target collision을 함께 봐야 한다. |
| SSD scale 식만 제시 | aspect ratio 1에는 인접 scale의 기하평균 $s'_k=\sqrt{s_ks_{k+1}}$인 추가 box가 있다. SSD300의 첫 `conv4_3` scale 같은 구현상 예외도 모델 명세에 기록해야 한다. |
| FPN을 $P_k=\mathrm{Conv}(\mathrm{lateral}(C_k)+\mathrm{up}(P_{k+1}))$로 표기 | canonical 계산을 분명히 하려면 재귀 중간값 $M_{k+1}$을 upsample하고, 각 $M_k$에 별도의 $3\times3$ convolution을 적용해 $P_k$를 만든다고 쓰는 편이 정확하다. |
| 마지막 $3\times3$ convolution이 노이즈를 제거 | 학습된 smoothing/output convolution이며 upsampling의 aliasing 영향을 줄이려는 설계다. 언제나 노이즈를 제거한다고 보장되지는 않는다. |
| YOLOv3 이후가 모두 FPN | 많은 계열이 feature pyramid 또는 PAN 계열 융합을 쓰지만 canonical FPN과 topology가 같다고 단정할 수 없다. 모델별 graph를 확인해야 한다. |
| Mask R-CNN은 FPN 없이는 불가능 | Mask R-CNN은 FPN과 잘 결합되지만 원 논문에는 단일-scale C4 backbone 실험도 있다. FPN은 강력한 선택이지 정의상 필수 조건은 아니다. |

또한 원본은 SSD를 2015년에 제안했다고 설명한다. preprint 시점을 가리키면 맞지만 널리 인용되는 학회 발표는 ECCV 2016이다. 날짜는 “논문 공개”와 “학회 발표”를 구분해서 적는다.

## 3. 작은 객체 병목을 수치로 보기

### 3.1 stride와 feature footprint

입력 크기가 $H\times W$이고 feature map이 $H_k\times W_k$이면 nominal stride는 다음처럼 볼 수 있다.

$$
D_{k,h}=\frac{H}{H_k},\qquad D_{k,w}=\frac{W}{W_k}
$$

SSD300의 `38 x 38` map에서 nominal stride는 약 $300/38\approx7.89$다. `1 x 1` map에서는 사실상 이미지 전체를 한 위치에서 요약한다. stride는 receptive field와 같지 않지만 위치 샘플링의 조밀함을 보여준다.

### 3.2 중심 양자화 오차

feature cell 중심만 후보 중심으로 쓴다고 단순화하면 한 축의 최대 양자화 오차는 대략 stride의 절반이다.

$$
e_{max}\approx\frac{D_k}{2}
$$

box offset regression이 이 오차를 보정하지만, 작은 객체는 몇 픽셀의 오차도 IoU를 크게 떨어뜨린다. 같은 중심을 가진 정사각형 box의 변 길이가 $q$이고 예측이 한 축으로 $\delta$만큼 이동했다고 하자. $0\le\delta<q$이면 IoU는 다음과 같다.

$$
\operatorname{IoU}=\frac{q(q-\delta)}{2q^2-q(q-\delta)}
=\frac{q-\delta}{q+\delta}
$$

$q=12$, $\delta=4$이면 IoU는 $8/16=0.5$다. 큰 객체에서는 같은 4픽셀 오차가 훨씬 덜 치명적이다.

### 3.3 grid collision metric

정답 중심을 feature map cell에 할당한 뒤 같은 cell의 객체 수를 센다. cell별 객체 수를 $n_{r,c}$라 하고 class 슬롯이 하나라면 최소 누락 수의 하한은 다음과 같다.

$$
N_{\mathrm{collision}}=\sum_{r,c}\max(0,n_{r,c}-1)
$$

학습 전에 해상도별 collision rate를 측정하면 어떤 prediction level이 필요한지 데이터로 결정할 수 있다.

## 4. SSD: 여러 feature map에 예측기를 복제한다

SSD300의 대표적인 prediction level은 다음과 같다. 정확한 layer 이름과 box 개수는 구현에 따라 달라질 수 있으므로 checkpoint의 config를 최종 계약으로 삼는다.

| level 예시 | 공간 shape | 셀당 box 예시 | 위치 수 | box 수 |
| --- | ---: | ---: | ---: | ---: |
| `conv4_3` | `38 x 38` | 4 | 1,444 | 5,776 |
| `conv7` | `19 x 19` | 6 | 361 | 2,166 |
| `conv8_2` | `10 x 10` | 6 | 100 | 600 |
| `conv9_2` | `5 x 5` | 6 | 25 | 150 |
| `conv10_2` | `3 x 3` | 4 | 9 | 36 |
| `conv11_2` | `1 x 1` | 4 | 1 | 4 |
| 합계 | - | - | 1,940 | 8,732 |

level $k$의 box 개수는 다음과 같다.

$$
B_k=H_kW_kA_k
$$

전체 default box 수는 다음과 같다.

$$
B_{\mathrm{all}}=\sum_{k=1}^{m}H_kW_kA_k
$$

### 4.1 prediction head shape

background를 별도 class로 포함한 softmax 방식이면 class 수는 $C+1$이다. level $k$의 convolution head는 다음 channel을 출력한다.

$$
K_{\mathrm{cls},k}=A_k(C+1),\qquad K_{\mathrm{box},k}=4A_k
$$

PyTorch raw output은 각각 `N x A_k(C+1) x H_k x W_k`, `N x 4A_k x H_k x W_k`다. 이를 다음 논리 shape로 바꾼다.

$$
\mathrm{cls}_k\in\mathbb{R}^{N\times H_kW_kA_k\times(C+1)}
$$

$$
\mathrm{box}_k\in\mathbb{R}^{N\times H_kW_kA_k\times4}
$$

모든 level을 box 축으로 concatenate하면 `N x 8732 x (C+1)`과 `N x 8732 x 4`가 된다.

## 5. default box를 만드는 수학

feature map이 $H_k\times W_k$일 때 cell $(r,c)$의 정규화 중심은 다음과 같다.

$$
d_x=\frac{c+0.5}{W_k},\qquad d_y=\frac{r+0.5}{H_k}
$$

scale $s_k$와 aspect ratio $a=w/h$를 사용하면 너비와 높이는 다음과 같다.

$$
d_w=s_k\sqrt{a},\qquad d_h=\frac{s_k}{\sqrt{a}}
$$

$m$개 level의 기본 scale을 선형 배치하는 대표 식은 다음과 같다.

$$
s_k=s_{\min}+\frac{s_{\max}-s_{\min}}{m-1}(k-1),qquad k=1,\ldots,m
$$

aspect ratio 1에는 같은 위치에 추가 box를 둘 수 있다.

$$
s'_k=\sqrt{s_ks_{k+1}}
$$

이 식만으로 실제 구현을 복원할 수는 없다. 마지막 level의 $s_{m+1}$, 첫 level의 특별 scale, aspect ratio별 중복 제거, 좌표 clip 여부를 config에 저장해야 한다.

### 5.1 수작업 예제

$2\times2$ feature map, $s=0.4$, $a=4$, cell $(r,c)=(0,1)$을 보자.

$$
d_x=\frac{1.5}{2}=0.75,\qquad d_y=\frac{0.5}{2}=0.25
$$

$$
d_w=0.4\sqrt4=0.8,\qquad d_h=\frac{0.4}{\sqrt4}=0.2
$$

따라서 `cxcywh`는 $(0.75,0.25,0.8,0.2)$다. `xyxy`로 바꾸면 $x_2=1.15$여서 이미지 밖으로 나간다. default box 생성 시 clip할지, decode 후 clip할지 정책을 반드시 고정해야 한다.

## 6. 매칭과 regression target

### 6.1 bipartite 성격의 강제 매칭

각 default box와 모든 정답 box 사이의 IoU를 계산한다. 보통 다음 두 규칙을 함께 쓴다.

1. 각 정답에 대해 IoU가 가장 큰 default box 하나는 threshold와 무관하게 양성으로 강제한다.
2. 나머지 default box는 최대 IoU가 threshold 이상이면 해당 정답의 양성으로 둔다.

첫 규칙이 없으면 아주 작은 정답이 어떤 default box와도 threshold를 넘지 못해 양성 표본이 0개가 될 수 있다. 같은 default box에 여러 정답이 경쟁하는 경우의 tie-breaking도 구현별로 고정해야 한다.

### 6.2 encode

scale variance를 $v_{xy}$, $v_{wh}$라 하면 대표적인 SSD target은 다음과 같다.

$$
t_x=\frac{g_x-d_x}{v_{xy}d_w},\qquad
t_y=\frac{g_y-d_y}{v_{xy}d_h}
$$

$$
t_w=\frac{\log(g_w/d_w)}{v_{wh}},\qquad
t_h=\frac{\log(g_h/d_h)}{v_{wh}}
$$

SSD 계열에서 흔한 값은 $v_{xy}=0.1$, $v_{wh}=0.2$지만 모델 config가 기준이다.

### 6.3 decode

예측 offset $\hat t$를 box로 되돌리는 식은 다음과 같다.

$$
\hat g_x=d_x+v_{xy}d_w\hat t_x,\qquad
\hat g_y=d_y+v_{xy}d_h\hat t_y
$$

$$
\hat g_w=d_w\exp(v_{wh}\hat t_w),\qquad
\hat g_h=d_h\exp(v_{wh}\hat t_h)
$$

`exp` 입력이 너무 크면 overflow가 난다. 추론 runtime과 학습 코드가 같은 clamp 범위를 적용하는지 확인한다.

## 7. NumPy로 수작업 값을 교차 검증하기

다음 코드는 **실행 가능한 검증용 예제**다. default box 생성, encode, decode round trip을 한 번에 검사한다.

```python
import math
import numpy as np


def make_default_boxes(height, width, scale, aspect_ratios):
    boxes = []
    for row in range(height):
        for col in range(width):
            cx = (col + 0.5) / width
            cy = (row + 0.5) / height
            for ratio in aspect_ratios:
                root = math.sqrt(ratio)
                boxes.append([cx, cy, scale * root, scale / root])
    return np.asarray(boxes, dtype=np.float32)


def encode(gt, default, variance_xy=0.1, variance_wh=0.2):
    target_xy = (gt[:2] - default[:2]) / (variance_xy * default[2:])
    target_wh = np.log(gt[2:] / default[2:]) / variance_wh
    return np.concatenate([target_xy, target_wh])


def decode(target, default, variance_xy=0.1, variance_wh=0.2):
    center = default[:2] + variance_xy * default[2:] * target[:2]
    size = default[2:] * np.exp(variance_wh * target[2:])
    return np.concatenate([center, size])


boxes = make_default_boxes(2, 2, 0.4, [1.0, 4.0])
assert boxes.shape == (8, 4)
np.testing.assert_allclose(boxes[3], [0.75, 0.25, 0.8, 0.2], atol=1e-7)

default = np.array([0.50, 0.50, 0.40, 0.20], dtype=np.float32)
ground_truth = np.array([0.54, 0.48, 0.44, 0.18], dtype=np.float32)
target = encode(ground_truth, default)
restored = decode(target, default)
np.testing.assert_allclose(restored, ground_truth, rtol=1e-6, atol=1e-6)
print("boxes", boxes.shape, "target", np.round(target, 6))
print("round_trip", np.round(restored, 6))
```

예상 target은 대략 $(1,-1,0.476551,-0.526803)$이다. 이 golden value는 Python, C++, C# 구현을 비교할 때 쓴다.

## 8. PyTorch로 SSD head shape 구현하기

다음 코드는 **실행 가능한 교육용 구현**이다. backbone, matcher, loss, NMS를 생략하고 prediction head의 channel-to-anchor 재배열만 검증한다.

```python
import torch
from torch import nn


class SSDPredictionHead(nn.Module):
    def __init__(self, in_channels, anchors_per_level, foreground_classes):
        super().__init__()
        self.class_count = foreground_classes + 1
        self.anchors_per_level = anchors_per_level
        self.cls_heads = nn.ModuleList()
        self.box_heads = nn.ModuleList()
        for channels, anchors in zip(in_channels, anchors_per_level):
            self.cls_heads.append(
                nn.Conv2d(channels, anchors * self.class_count, kernel_size=3, padding=1)
            )
            self.box_heads.append(
                nn.Conv2d(channels, anchors * 4, kernel_size=3, padding=1)
            )

    @staticmethod
    def _reshape(raw, anchors, values_per_anchor):
        batch, _, height, width = raw.shape
        logical = raw.reshape(batch, anchors, values_per_anchor, height, width)
        logical = logical.permute(0, 3, 4, 1, 2).contiguous()
        return logical.reshape(batch, height * width * anchors, values_per_anchor)

    def forward(self, features):
        class_outputs = []
        box_outputs = []
        for feature, cls_head, box_head, anchors in zip(
            features, self.cls_heads, self.box_heads, self.anchors_per_level
        ):
            class_outputs.append(
                self._reshape(cls_head(feature), anchors, self.class_count)
            )
            box_outputs.append(self._reshape(box_head(feature), anchors, 4))
        return torch.cat(class_outputs, dim=1), torch.cat(box_outputs, dim=1)


torch.manual_seed(7)
features = [
    torch.randn(2, 32, 8, 8),
    torch.randn(2, 64, 4, 4),
    torch.randn(2, 128, 2, 2),
]
head = SSDPredictionHead([32, 64, 128], [4, 6, 6], foreground_classes=3)
class_logits, box_offsets = head(features)
expected_boxes = 8 * 8 * 4 + 4 * 4 * 6 + 2 * 2 * 6
assert expected_boxes == 376
assert class_logits.shape == (2, 376, 4)
assert box_offsets.shape == (2, 376, 4)
loss = class_logits.square().mean() + box_offsets.square().mean()
loss.backward()
assert all(parameter.grad is not None for parameter in head.parameters())
print(class_logits.shape, box_offsets.shape, f"loss={loss.item():.6f}")
```

## 9. FPN: 의미를 고해상도 쪽으로 전달한다

### 9.1 네 단계

1. bottom-up backbone이 `C3`, `C4`, `C5`를 만든다.
2. 각 `Ck`에 `1x1 Conv`를 적용해 channel을 같은 $F$로 맞춘다.
3. 깊은 중간 feature를 upsample하여 현재 lateral feature와 원소별로 더한다.
4. 각 중간 feature에 `3x3 Conv`를 적용해 `Pk`를 출력한다.

canonical 계산을 중간 feature $M_k$로 쓰면 다음과 같다.

$$
M_5=\operatorname{Conv}_{1\times1}(C_5)
$$

$$
M_k=\operatorname{Conv}_{1\times1}(C_k)
+\operatorname{Upsample}(M_{k+1}),\qquad k=4,3
$$

$$
P_k=\operatorname{Conv}_{3\times3}(M_k)
$$

덧셈하려면 spatial shape와 channel 수가 모두 같아야 한다. 입력 크기가 홀수이면 단순 `scale_factor=2`가 lateral shape와 한 칸 어긋날 수 있으므로 목표 `size`를 직접 지정한다.

### 9.2 shape 추적

입력 `N x 3 x 256 x 320`에서 backbone stride가 각각 8, 16, 32라면 다음 예를 얻는다.

| tensor | bottom-up shape | lateral 뒤 | top-down 결합 뒤 | 최종 출력 |
| --- | --- | --- | --- | --- |
| `C3` | `N x 128 x 32 x 40` | `N x 64 x 32 x 40` | `M3: N x 64 x 32 x 40` | `P3: N x 64 x 32 x 40` |
| `C4` | `N x 256 x 16 x 20` | `N x 64 x 16 x 20` | `M4: N x 64 x 16 x 20` | `P4: N x 64 x 16 x 20` |
| `C5` | `N x 512 x 8 x 10` | `N x 64 x 8 x 10` | `M5: N x 64 x 8 x 10` | `P5: N x 64 x 8 x 10` |

## 10. PyTorch FPN 최소 구현

다음 코드는 **실행 가능한 교육용 구현**이다. `C3`, `C4`, `C5`가 이미 준비되었다고 가정하며, odd spatial shape도 처리한다.

```python
import torch
from torch import nn
from torch.nn import functional as F


class TinyFPN(nn.Module):
    def __init__(self, in_channels, out_channels):
        super().__init__()
        self.lateral = nn.ModuleList(
            nn.Conv2d(channels, out_channels, kernel_size=1)
            for channels in in_channels
        )
        self.output = nn.ModuleList(
            nn.Conv2d(out_channels, out_channels, kernel_size=3, padding=1)
            for _ in in_channels
        )

    def forward(self, features):
        if len(features) != len(self.lateral):
            raise ValueError("feature level count mismatch")
        middle = [None] * len(features)
        middle[-1] = self.lateral[-1](features[-1])
        for index in range(len(features) - 2, -1, -1):
            side = self.lateral[index](features[index])
            top = F.interpolate(middle[index + 1], size=side.shape[-2:], mode="nearest")
            middle[index] = side + top
        return [layer(value) for layer, value in zip(self.output, middle)]


torch.manual_seed(11)
inputs = [
    torch.randn(2, 128, 33, 41),
    torch.randn(2, 256, 17, 21),
    torch.randn(2, 512, 9, 11),
]
fpn = TinyFPN([128, 256, 512], 64)
outputs = fpn(inputs)
assert [tuple(x.shape) for x in outputs] == [
    (2, 64, 33, 41),
    (2, 64, 17, 21),
    (2, 64, 9, 11),
]
sum(x.mean() for x in outputs).backward()
assert all(parameter.grad is not None for parameter in fpn.parameters())
print([tuple(x.shape) for x in outputs])
```

## 11. C++ default box와 round-trip 검증

다음 코드는 외부 library가 필요 없는 **실행 가능한 C++17 예제**다. model inference 대신 Python과 같은 box 계약을 검증한다.

```cpp
#include <array>
#include <cassert>
#include <cmath>
#include <iomanip>
#include <iostream>
#include <vector>

using Box = std::array<float, 4>;

std::vector<Box> MakeDefaultBoxes(
    int height, int width, float scale, const std::vector<float>& ratios) {
    std::vector<Box> boxes;
    for (int row = 0; row < height; ++row) {
        for (int col = 0; col < width; ++col) {
            const float cx = (static_cast<float>(col) + 0.5F) / width;
            const float cy = (static_cast<float>(row) + 0.5F) / height;
            for (float ratio : ratios) {
                const float root = std::sqrt(ratio);
                boxes.push_back({cx, cy, scale * root, scale / root});
            }
        }
    }
    return boxes;
}

Box Encode(const Box& gt, const Box& d, float vxy = 0.1F, float vwh = 0.2F) {
    return {
        (gt[0] - d[0]) / (vxy * d[2]),
        (gt[1] - d[1]) / (vxy * d[3]),
        std::log(gt[2] / d[2]) / vwh,
        std::log(gt[3] / d[3]) / vwh,
    };
}

Box Decode(const Box& t, const Box& d, float vxy = 0.1F, float vwh = 0.2F) {
    return {
        d[0] + vxy * d[2] * t[0],
        d[1] + vxy * d[3] * t[1],
        d[2] * std::exp(vwh * t[2]),
        d[3] * std::exp(vwh * t[3]),
    };
}

int main() {
    const auto boxes = MakeDefaultBoxes(2, 2, 0.4F, {1.0F, 4.0F});
    assert(boxes.size() == 8);
    assert(std::abs(boxes[3][0] - 0.75F) < 1e-6F);
    assert(std::abs(boxes[3][2] - 0.80F) < 1e-6F);

    const Box d{0.50F, 0.50F, 0.40F, 0.20F};
    const Box gt{0.54F, 0.48F, 0.44F, 0.18F};
    const Box target = Encode(gt, d);
    const Box restored = Decode(target, d);
    for (int index = 0; index < 4; ++index) {
        assert(std::abs(restored[index] - gt[index]) < 1e-6F);
    }
    std::cout << std::fixed << std::setprecision(6)
              << target[0] << " " << target[1] << " "
              << target[2] << " " << target[3] << "\n";
}
```

## 12. C# default box와 round-trip 검증

다음 코드는 **실행 가능한 C# 예제**다. Python·C++과 같은 입력과 허용 오차를 사용한다.

```csharp
using System;
using System.Collections.Generic;

public static class SsdBoxDemo
{
    public static List<float[]> MakeDefaultBoxes(
        int height, int width, float scale, float[] ratios)
    {
        var boxes = new List<float[]>();
        for (int row = 0; row < height; row++)
        {
            for (int col = 0; col < width; col++)
            {
                float cx = (col + 0.5f) / width;
                float cy = (row + 0.5f) / height;
                foreach (float ratio in ratios)
                {
                    float root = (float)Math.Sqrt(ratio);
                    boxes.Add(new[] { cx, cy, scale * root, scale / root });
                }
            }
        }
        return boxes;
    }

    public static float[] Encode(float[] gt, float[] d)
    {
        const float varianceXy = 0.1f;
        const float varianceWh = 0.2f;
        return new[]
        {
            (gt[0] - d[0]) / (varianceXy * d[2]),
            (gt[1] - d[1]) / (varianceXy * d[3]),
            (float)(Math.Log(gt[2] / d[2]) / varianceWh),
            (float)(Math.Log(gt[3] / d[3]) / varianceWh),
        };
    }

    public static float[] Decode(float[] target, float[] d)
    {
        const float varianceXy = 0.1f;
        const float varianceWh = 0.2f;
        return new[]
        {
            d[0] + varianceXy * d[2] * target[0],
            d[1] + varianceXy * d[3] * target[1],
            d[2] * (float)Math.Exp(varianceWh * target[2]),
            d[3] * (float)Math.Exp(varianceWh * target[3]),
        };
    }

    public static void Main()
    {
        List<float[]> boxes = MakeDefaultBoxes(2, 2, 0.4f, new[] { 1.0f, 4.0f });
        if (boxes.Count != 8 || Math.Abs(boxes[3][2] - 0.8f) > 1e-6f)
            throw new Exception("default box mismatch");

        float[] d = { 0.50f, 0.50f, 0.40f, 0.20f };
        float[] gt = { 0.54f, 0.48f, 0.44f, 0.18f };
        float[] target = Encode(gt, d);
        float[] restored = Decode(target, d);
        for (int index = 0; index < 4; index++)
            if (Math.Abs(restored[index] - gt[index]) > 1e-6f)
                throw new Exception("round trip mismatch");

        Console.WriteLine(
            string.Format("{0:F6} {1:F6} {2:F6} {3:F6}",
                target[0], target[1], target[2], target[3]));
    }
}
```

## 13. 프레임워크 간 shape·layout·dtype 계약

| 단계 | PyTorch Python | C++ runtime 예시 | C# runtime 예시 | 주의점 |
| --- | --- | --- | --- | --- |
| 입력 image | `NCHW`, `float32` | engine contract에 따라 `NCHW` 또는 `NHWC` | tensor API에 따라 다름 | resize, normalization, RGB/BGR까지 schema화한다. |
| raw class | `N x A(C+1) x H x W` | export graph의 실제 layout | export graph의 실제 layout | reshape 전에 anchor-major channel 순서를 확인한다. |
| 논리 class | `N x B x (C+1)` | contiguous flat buffer + stride | multidimensional 또는 flat array | background index를 고정한다. |
| raw box | `N x 4A x H x W` | `float` 또는 half | `float` | channel 순서가 anchor별 `dx,dy,dw,dh`인지 확인한다. |
| default box | `B x 4`, `float32` | `std::vector<Box>` | `List<float[]>` 또는 tensor | 생성 순서가 level→row→col→ratio인지 고정한다. |
| FPN feature | `NCHW` | 보통 opaque engine tensor | 보통 runtime tensor | level 이름과 stride를 같이 저장한다. |

`permute` 뒤 `reshape`는 memory layout을 바꿀 수 있다. Python에서 우연히 맞는 flat 순서를 C++에서 추측하지 말고, 작은 tensor에 연속 정수를 넣은 index-map golden test를 만든다.

`float16`은 convolution에 유리하지만 default box 생성과 decode의 `exp`, IoU, NMS는 `float32`가 안전한 경우가 많다. mixed precision 경계는 backend별로 검증한다.

## 14. 테스트 전략

### 14.1 default box 단위 테스트

- level별 생성 개수가 $H_kW_kA_k$와 같은가?
- 첫 box, 마지막 box, 이미지 밖으로 나가는 box가 기대값과 같은가?
- aspect ratio $a$와 $1/a$의 너비·높이가 서로 바뀌는가?
- box 생성 순서가 head flatten 순서와 같은가?
- config가 바뀌면 golden hash가 의도적으로 바뀌는가?

### 14.2 encode/decode 속성 테스트

유효한 양의 너비·높이에 대해 다음 round trip을 검사한다.

$$
\operatorname{decode}(\operatorname{encode}(g,d),d)\approx g
$$

random box를 만들 때 0 또는 음수 크기는 제외한다. 극단적으로 작은 $d_w,d_h$는 별도 경계 테스트로 분리한다.

### 14.3 matcher 테스트

- IoU threshold 바로 아래·같음·바로 위를 모두 검사한다.
- 모든 IoU가 threshold 미만이어도 각 정답이 최소 한 양성을 얻는지 검사한다.
- 정답 0개인 이미지에서 모든 default box가 background가 되는지 검사한다.
- crowd·ignore box가 background hard negative로 잘못 들어가지 않는지 검사한다.

### 14.4 FPN 테스트

- 홀수 spatial shape에서 lateral과 top-down 덧셈이 성공하는가?
- 각 output channel이 동일한가?
- 모든 lateral·output convolution에 gradient가 도달하는가?
- level 순서를 뒤집으면 조용히 통과하지 않고 실패하는가?

## 15. 디버깅 지도

| 증상 | 우선 확인할 것 | 대표 원인 |
| --- | --- | --- |
| box가 규칙적인 격자로 흩어짐 | decode 전후 중심 좌표 | offset variance 누락 또는 cell 중심 0.5 누락 |
| 너비·높이가 지나치게 큼 | `tw,th`와 `exp` 입력 | variance 중복 적용, clamp 누락 |
| class는 맞지만 box가 다른 위치 | anchor flatten index | head와 default box의 level·row·col·ratio 순서 불일치 |
| 작은 객체 recall이 0에 가까움 | level별 positive count | 작은 scale 누락, matcher threshold 과대, resize로 객체 소실 |
| FPN 덧셈 shape error | odd input의 각 spatial shape | `scale_factor=2`만 사용 |
| 학습 loss는 감소하지만 추론 결과 없음 | background index와 score activation | softmax/sigmoid 계약 혼동 |
| 특정 backend만 결과가 다름 | layout과 dtype | `NCHW`/`NHWC` 오해, half `exp` overflow |

디버깅할 때 최종 mAP부터 보지 않는다. `level별 default box 수 → positive match 수 → raw offset → decoded box → clip → score → NMS` 순서로 중간 artifact를 비교한다.

## 16. 성능·메모리·수치 안정성

### 16.1 prediction tensor 메모리

SSD300, VOC의 $B=8732$, $C=20$, batch $N=32$에서 logical class와 box tensor의 원소 수는 다음과 같다.

$$
M=32\times8732\times((20+1)+4)=6{,}985{,}600
$$

`float32`만 세어도 다음 byte가 필요하다.

$$
4M=27{,}942{,}400\ \mathrm{bytes}\approx26.65\ \mathrm{MiB}
$$

학습에서는 gradient, activation, matcher 임시 tensor가 추가된다. 모든 정답과 모든 default box의 IoU matrix를 batch 최대 정답 수로 padding하면 낭비가 커질 수 있다.

### 16.2 FPN 메모리

고해상도 `P3`가 메모리를 지배하기 쉽다. level $k$의 feature byte는 대략 다음과 같다.

$$
\mathrm{bytes}_k=N\times F\times H_k\times W_k\times\mathrm{dtypeBytes}
$$

작은 객체 recall을 높이려고 더 얕은 `P2`를 추가하면 latency보다 memory bandwidth와 activation memory가 먼저 병목이 될 수 있다.

### 16.3 수치 안정성

- encode 전에 $g_w,g_h,d_w,d_h>0$을 assert한다.
- decode의 `exp` 입력을 검증된 범위로 clamp한다.
- IoU의 union 분모에는 작은 epsilon을 둔다.
- `float16`에서 큰 음수 logit의 softmax와 큰 `tw,th`를 따로 시험한다.
- NMS 직전 NaN과 무한대를 제거하고 발생 count를 metric으로 남긴다.

### 16.4 실제 성능 측정

kernel latency뿐 아니라 preprocessing, host-device copy, decode, top-k, NMS를 포함한 end-to-end p50/p95/p99를 잰다. 입력의 객체 수와 score threshold에 따라 NMS 비용이 달라지므로 빈 이미지와 혼잡 이미지를 모두 benchmark한다.

## 17. 실무 실패 사례

### 사례 A: anchor 순서가 한 칸씩 밀림

학습 head는 `row → col → ratio` 순서인데 배포 decoder는 `ratio → row → col` 순서로 default box를 만들었다. tensor shape와 원소 수는 모두 맞아 오류가 나지 않지만 box가 규칙적으로 엉뚱한 위치에 생긴다.

대응은 연속 index를 넣은 가짜 head와 첫·중간·마지막 anchor golden test를 Python·배포 runtime에서 공유하는 것이다.

### 사례 B: letterbox 이후 작은 객체가 label에서 사라짐

이미지를 축소한 뒤 폭이나 높이가 1픽셀 미만인 box를 data pipeline이 버렸다. 전체 loss는 안정적으로 줄지만 작은 객체 recall은 회복되지 않는다.

대응은 resize 전후 box 크기 histogram, drop reason, class별 drop rate를 기록하는 것이다. 작은 객체가 핵심이면 crop·tiling과 입력 해상도도 함께 검토한다.

### 사례 C: FPN level 의미가 export에서 뒤바뀜

Python dictionary의 `P3,P4,P5`를 이름이 아닌 순회 순서에 의존해 export했고 C++은 다른 순서로 head를 연결했다. 각 tensor channel이 같아 shape 검사만으로 발견하지 못했다.

대응은 output name에 level을 넣고, stride와 spatial shape를 manifest에 기록하며, content fingerprint test를 둔다.

### 사례 D: `scale_factor=2`가 홀수 입력에서 실패

입력 크기가 고정된 학습에서는 우연히 맞았지만 동적 해상도 배포에서 `C4`가 `17 x 21`, upsample 결과가 `18 x 22`가 되어 lateral addition이 실패했다.

대응은 lateral tensor의 `shape[-2:]`를 target size로 사용하고 허용 입력 shape 전체를 export test에 포함하는 것이다.

### 사례 E: training과 inference의 variance 불일치

학습은 $(0.1,0.2)$로 encode했지만 배포 decoder는 variance를 1로 두었다. 중심은 크게 튀고 크기는 `exp` 때문에 폭발했다.

대응은 variance를 model artifact의 schema에 넣고 round-trip golden vector를 배포 시작 시 검사하는 것이다.

## 21. 해답

### 해답 1

box 수는 다음과 같다.

$$
20\times20\times6=2400
$$

background를 포함한 class 수가 5이므로 raw class channel 수는 다음과 같다.

$$
6\times(4+1)=30
$$

### 해답 2

$$
d_x=\frac{1+0.5}{4}=0.375,\qquad
d_y=\frac{2+0.5}{4}=0.625
$$

### 해답 3

$$
d_w=0.3\sqrt9=0.9,\qquad
d_h=\frac{0.3}{\sqrt9}=0.1
$$

### 해답 4

$$
t_x=\frac{0.54-0.50}{0.1\times0.4}=1
$$

$$
t_y=\frac{0.48-0.50}{0.1\times0.2}=-1
$$

### 해답 5

단순 2배 결과는 `34 x 42`여서 `33 x 41` lateral과 spatial shape가 다르다. `interpolate(..., size=lateral.shape[-2:])`처럼 목표 크기를 직접 지정한다.

### 해답 6

첫째 level 순서, 둘째 각 level의 `row → col → ratio` flatten 순서, 셋째 box channel의 `dx,dy,dw,dh` 배치를 비교한다. 그다음 scale·aspect ratio config와 center offset 0.5를 확인한다.

### 해답 7

SSD는 여러 해상도 feature map에서 직접 예측해 객체 크기별 head 위치를 분담한다. FPN은 깊은 의미 feature를 위에서 아래로 전달해 여러 해상도의 표현력을 강화한다.

## 핵심 요약

- 작은 객체 문제는 단순 receptive field 크기가 아니라 stride, downsampling 정보 손실, target collision이 함께 만든다.
- SSD는 여러 feature map마다 class·box head를 붙여 multi-scale prediction을 수행한다.
- level별 box 수는 $H_kW_kA_k$이고 head flatten 순서와 default box 생성 순서가 반드시 같아야 한다.
- default box는 cell 중심, scale, aspect ratio로 만들며 aspect ratio 1의 추가 기하평균 scale 같은 구현 세부가 있다.
- matcher는 각 정답에 최소 한 positive를 강제해 작은 객체가 학습에서 완전히 사라지는 일을 막는다.
- encode와 decode의 variance는 model contract이며 `exp`와 0 크기에 수치 안정성 처리가 필요하다.
- FPN은 lateral projection, top-down upsample, element-wise addition, output convolution으로 깊은 의미와 고해상도를 결합한다.
- canonical FPN의 재귀 중간값 $M_k$와 최종 output $P_k$를 구분하면 구현과 수식이 명확해진다.
- odd·dynamic input에서는 2배 배율이 아니라 lateral의 정확한 spatial size로 upsample한다.
- 배포 장애는 모델보다 anchor ordering, layout, variance, preprocessing, level naming 계약 불일치에서 자주 생긴다.
